## BUG-001: 사진 일괄 선택 불가

**현상**: 사진을 하나씩 클릭해서 넣어야 함. 5장을 넣으려면 5번 반복.

**기대 동작**:
- `[ + 사진 고르기 ]` 탭 → 네이티브 갤러리에서 **최대 5장 한꺼번에 선택**
- `<input type="file" multiple accept="image/*">`
- 선택 후 즉시 그리드에 모든 사진 표시
- 이미 N장 있으면 `5 - N`장까지만 추가 선택 가능

**수정 포인트**:
- 파일 입력에 `multiple` 속성 추가
- 선택된 파일 배열을 한꺼번에 처리
- 5장 초과 선택 시 앞에서 5장만 취하고 토스트 알림

## BUG-002: 사진 용량 제한 없음

**현상**: 고해상도 사진 (iPhone 48MP 등) 올리면 화면이 멈추거나 다음 단계로 넘어가지 않음.

**기대 동작**:
- 사진 선택 즉시 클라이언트에서 리사이즈
- 사용자는 리사이즈를 인지하지 못함 (자동 처리)

**수정 포인트**:
```
사진 선택
  ↓
Canvas API로 리사이즈 (max 1920px 긴 변 기준)
  ↓
JPEG quality 0.8로 변환
  ↓
5MB 초과 → quality 0.6으로 재압축
  ↓
그래도 초과 → max 1280px로 추가 리사이즈
  ↓
HEIC/HEIF → JPEG 변환 (heic2any)
```

**성능 기준**: 사진 1장 리사이즈 < 1초

## BUG-003: 에센스 결과 → 채팅 전환 안 됨

**현상**: 에센스 결과 화면에서 질문에 답한 후, 바로 채팅으로 안 넘어감. 한 번 더 보내야 채팅이 시작됨.

**기대 동작**:
```
에센스 결과 화면
  ↓
사용자가 답 입력 + [답하기] 탭
  ↓
즉시 /chat으로 전환
  ↓
채팅 화면에 이미 표시됨:
  [AI] 에센스 요약 + 첫 질문
  [사용자] 방금 입력한 답
  [AI 타이핑 중...]
  ↓
두 번째 턴 AI 응답 도착 → 표시
  ↓
이후 일반 채팅처럼 자유롭게 주고받기
```

**수정 포인트**:
- 에센스 결과 화면의 응답을 첫 번째 사용자 메시지로 세션에 저장
- `/chat` 전환 시 세션 ID를 URL에 포함
- 채팅 화면 마운트 시 기존 메시지 로드 + 자동으로 두 번째 턴 API 호출
- **전환 후 입력 영역이 즉시 활성화**되어 추가 메시지 전송 가능

## BUG-004: 대화 중 사진 입력 후 다음 대화 불가

**현상**: 채팅 중 사진을 보내면 AI가 응답한 후, 다음 메시지를 보낼 수 없음.

**기대 동작**:
- 사진 전송 → AI 응답 → 입력 영역 다시 활성화 → 텍스트든 사진이든 계속 보낼 수 있음
- 사진 전송 후 상태:
  - `isSending: false`로 리셋
  - 입력 필드 클리어
  - `[ → ]` 버튼 활성화 (입력 있을 때)
  - `📷` 버튼 활성화

**수정 포인트**:
- API 응답 수신 후 `isSending` 상태 리셋 확인
- 사진 전송 시 사용하는 상태 변수가 텍스트 전송과 다른 경우 통합
- 에러 발생 시에도 상태 리셋 보장 (`finally` 블록)

```typescript
const sendMessage = async (content, type) => {
  try {
    setIsSending(true);
    // ... API 호출
  } catch (error) {
    // 에러 처리
  } finally {
    setIsSending(false);  // 반드시 리셋
    setAttachedPhotos([]);  // 첨부 사진 클리어
    inputRef.current?.focus();  // 입력 필드 포커스
  }
};
```

## BUG-005: 오타 지적 톤이 불쾌함

**현상**: AI가 오타를 지적하는데, 꼽주는 느낌이 남.

**기대 동작**:
- **오타를 절대 지적하지 않는다**
- 사용자가 쓴 표현을 그대로 받아들인다
- 맞춤법, 띄어쓰기, 오타 모두 무시
- 사용자의 말을 AI 언어로 바꾸지 않는다 (기존 원칙 5번인데 지켜지지 않고 있음)

**수정 포인트**:
- 시스템 프롬프트에 명시적으로 추가:

```
## 절대 하지 않는 것 (추가)
6. 사용자의 오타, 맞춤법, 띄어쓰기를 지적하지 않는다
7. 사용자의 표현을 "정확한" 표현으로 바꾸지 않는다
8. 사용자가 쓴 단어를 그대로 사용한다 ("갈리다"를 "갈리다"로, "이상하다"를 "이상하다"로)
```

- 톤 전체 점검: "꼽주는" 느낌의 근원
  - 확인 질문이 교정처럼 느껴지는 경우: "~라고 하셨는데, ~인 거죠?" → 삭제
  - 사용자 말을 반복할 때 미세하게 고쳐서 반복하는 경우 → 그대로 인용

## BUG-006: 질문만 반복, 분석/발견 안 함

**현상**: AI가 계속 질문만 함. 가설도 안 던지고, 발견도 안 만들고, 연결도 안 함. 루즈함.

**기대 동작** (메타다이얼로그 코어 로직):

### 대화 패턴 — 질문만이 아니라 3가지를 섞는다

| 턴 유형 | 비율 | 예시 |
|---------|------|------|
| **질문** (탐색) | ~40% | "이 사진에서 뭐가 끌려?" |
| **가설** (던지기) | ~30% | "혹시 넓은 곳보다 손 닿는 곳이 편한 건 아닌가요?" |
| **연결/발견** (분석) | ~30% | "전에 '혼자 채우는 패턴' 얘기했는데, 이것도 같은 맥락 같아요" |

### 턴별 전략 (구체적)

```
턴 1: 열린 질문 (관찰 기반)
턴 2: 사용자 응답 기반 가설 (틀려도 됨)
턴 3: 가설 반응 + 더 깊은 질문
턴 4: 다른 각도에서 던지기 OR 에센스 연결
턴 5: 여기까지의 대화에서 패턴/발견 시도
턴 6+: 발견이 있으면 인사이트 카드. 없으면 새 각도
턴 8-10: 자연스러운 정리. "오늘 이런 게 보였어요"
```

### 금지 패턴

| 이렇게 하면 안 됨 | 이렇게 해야 함 |
|------------------|---------------|
| 연속 3턴 질문만 | 2턴 질문 후 반드시 가설 or 연결 |
| "그래요?", "더 이야기해줄래요?" | 구체적: "그때 어떤 느낌이었어요?" |
| "좋은 관점이네요" | 연결: "전에 이런 얘기도 했는데..." |
| 매 턴 에센스 언급 | 세션당 1-2회만. 자연스러울 때만 |
| 사용자 말 요약만 | 재프레이밍: "그러니까 X보다 Y가 중요한 거네요?" |

### 발견 감지 기준

다음 중 하나가 발생하면 인사이트 카드 표시:
1. 사용자가 "아, 그래서 그랬구나" 류의 깨달음 표현
2. 이전 대화와 모순되는 반응 (패턴 깨짐)
3. 감각 에센스 → 행동 패턴으로 연결이 성립
4. 사용자가 글의 언어를 자기 언어로 전환하는 순간
5. AI가 3턴 이상 같은 주제를 파고들었을 때 → 정리 시도

**수정 포인트**:
- `engine/dialogue.md` 시스템 프롬프트 전면 리뷰
- 턴 카운터 추가: 현재 몇 턴째인지 프롬프트에 전달
- 대화 히스토리에서 "질문만 연속 N회" 감지 → 프롬프트에 "가설을 던져라" 지시
- 인사이트 감지 로직 강화: 5턴 이상 대화 시 반드시 1회 이상 발견 시도

## BUG-007: 탭 이동 후 채팅 세션 사라짐

**현상**: 채팅 중 하단 탭바에서 다른 탭(세션, 프로필) 클릭 후 다시 채팅으로 돌아오면 세션이 없어져 있음.

**기대 동작**:
- 탭 이동해도 현재 채팅 세션 유지
- 채팅 탭(홈)으로 돌아왔을 때:
  - 진행 중인 세션이 있으면 → 해당 세션 대화 화면 표시
  - 없으면 → 홈 화면 (기본)

**수정 포인트**:

```
방안 1: URL 기반 세션 유지
  /chat?session=abc123 형태로 세션 ID를 URL에 유지
  탭 전환은 라우팅이므로 히스토리에 남음
  뒤로가기/앞으로가기로 복원 가능

방안 2: 전역 상태 (Context/Zustand)
  activeSessionId를 전역 상태로 관리
  /home 진입 시 activeSessionId 체크 → 있으면 "이어서 이야기할까요?" 배너

방안 3: (권장) 둘 다
  URL에 세션 ID + 전역 상태에도 유지
  탭바에서 "홈" 탭 = /home (배너로 세션 복귀 유도)
  탭바에 "채팅" 탭이 별도로 있으면 = /chat?session=current
```

**추가 고려**:
- 채팅 중에는 하단 탭바를 숨기는 것도 방법 (현재 스펙에 이미 반영)
- 숨기면 이 버그 자체가 발생하지 않음
- 단, `/chat`에서 나가려면 헤더의 `← 홈` 사용

## BUG-008: 주간 리포트 생성 불가

**현상**: 프로필에서 주간 리포트가 생성되지 않음.

**확인 필요**:
- 주간 리포트 기능이 현재 스펙에 포함되어 있는가?
- 기존 스펙에는 "주간 리포트"가 없음 → **새 기능으로 정의 필요**

### 주간 리포트 기능 정의 (신규)

**목적**: 한 주간의 대화에서 발견된 패턴과 변화를 정리

**생성 조건**:
- 해당 주에 세션 1개 이상 존재
- 일요일 자정 자동 생성 또는 사용자 수동 트리거

**리포트 내용**:
```
┌──────────────────────────────────────┐
│  3월 4주차 (3.24 ~ 3.29)            │
│                                      │
│  이번 주 대화 3회                     │
│                                      │
│  새로운 발견                          │
│  · "넓은 곳이 필요한 상태"            │
│  · "닫는 것에 대한 두려움"            │
│                                      │
│  반복된 키워드                        │
│  #공간  #통제  #속도                  │
│                                      │
│  에센스 변화                          │
│  (변화 없음 / "양감이 조금 넓어졌어요")│
│                                      │
│  한 줄 요약                           │
│  "이번 주는 공간과 통제에 대한         │
│   이야기가 많았어요"                   │
└──────────────────────────────────────┘
```

**위치**: `/profile` 하단, "이번 주" 섹션
**인터랙션**: 카드 탭 → 상세 바텀시트 (세션별 링크 포함)
**API**: `GET /api/weekly-report?week=2026-W13`
