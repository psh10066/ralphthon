# 랄프톤 — 화면 설계

디자인 톤: 투명, 유리, 여백, 거울. 군더더기 없음. 콘텐츠가 장식.

모바일 퍼스트 375px, 데스크톱 max-width 480px 센터 정렬.

---

## `/` — 원페이지 퍼널

### 랜딩 (뷰포트 1)

```
┌──────────────────────────────────────┐
│                                      │
│     사진 5장이면                      │
│     당신이 보입니다.                  │
│                                      │
│         [ 시작하기 ]                  │
│                                      │
│  ┌────────┐ ┌────────┐ ┌────────┐   │
│  │ 예시1  │ │ 예시2  │ │ 예시3  │   │
│  │ 에센스 │ │ 에센스 │ │ 에센스 │   │
│  │ 카드   │ │ 카드   │ │ 카드   │   │
│  └────────┘ └────────┘ └────────┘   │
│       ← 가로 스크롤 →                │
└──────────────────────────────────────┘
```

### 업로드 (뷰포트 2)

```
┌──────────────────────────────────────┐
│                                      │
│   끌리는 사진으로.                    │
│   예쁠 필요 없어요.                   │
│                                      │
│       [ + 사진 고르기 ]               │  ← 0장일 때: 큰 버튼
│                                      │
│   ┌─────┐ ┌─────┐ ┌─────┐          │  ← 선택 후: 그리드
│   │ 📷1 │ │ 📷2 │ │ 📷3 │          │
│   └─────┘ └─────┘ └─────┘          │
│       ┌─────┐ ┌─────┐              │
│       │ 📷4 │ │  +  │     4/5      │
│       └─────┘ └─────┘              │
│                                      │
│        [ 나를 읽어줘 ]                │  ← 5장 미만: 비활성 (회색)
└──────────────────────────────────────┘
```

#### 업로드 방식 (BUG-001 수정)

| 방식 | 동작 |
|------|------|
| **갤러리 멀티셀렉트** | `[ + 사진 고르기 ]` 탭 → `<input type="file" multiple accept="image/*">` → 최대 5장 한꺼번에 선택 |
| **드래그 앤 드롭** | 데스크톱에서 파일 드래그 → 드롭존 하이라이트 |
| **카메라 촬영** | `[ 📷 ]` 버튼 → 1장씩 추가 |
| **붙여넣기** | Ctrl+V → 이미지 자동 감지 추가 |

- 5장 초과 선택 시: 앞에서 5장만 취함 + 토스트 "5장까지 가능해요"
- 5장 미만: `[ 나를 읽어줘 ]` 비활성 (회색, 탭 불가)
- 각 사진 우상단 `[ × ]`: 제거. 나머지 앞으로 당김
- 사진 길게 누르기: 드래그로 순서 변경

#### 이미지 자동 리사이즈 (BUG-002 수정)

```
사진 선택 즉시:
  Canvas API → max 1920px 리사이즈 → JPEG 0.8
  5MB 초과 → quality 0.6 재압축
  그래도 초과 → max 1280px 추가 리사이즈
  HEIC/HEIF → JPEG 변환 (heic2any)
  실패 → 토스트 "이 사진은 올릴 수 없어요" + 슬롯 비움
```

사용자는 리사이즈를 인지하지 못한다. 1장당 < 1초.

#### 업로드 엣지케이스

| 케이스 | 처리 |
|--------|------|
| GIF 업로드 | 첫 프레임만 추출 |
| 동영상 선택 | 무시 + 토스트 "사진만 올릴 수 있어요" |
| 갤러리 권한 거부 | 토스트 + 설정 딥링크 |
| 오프라인 | 사진 선택은 가능 (로컬). 분석 버튼만 비활성 |
| 페이지 새로고침 | 선택한 사진 유실 (복구 불가) |
| 브라우저 메모리 부족 | 리사이즈 먼저 → 원본 즉시 해제 |

### 로딩 → 거울 → 대화 전환

로딩 완료 후 에센스 결과가 나오면, 하단에 관찰 + 열린 질문이 표시된다.
사용자가 자유 응답을 입력하면 → 즉시 `/chat`으로 전환.
에센스 결과 화면은 "끝"이 아니라 "첫 대화 턴"이다.

### 로딩

```
┌──────────────────────────────────────┐
│                                      │
│         당신을 읽고 있어요...         │
│                                      │
│   ✓ 색감을 보고 있어요               │
│   ✓ 질감을 느끼고 있어요             │
│   ● 온도를 재고 있어요               │
│   ○ 언어로 옮기고 있어요             │
│                                      │
└──────────────────────────────────────┘
```

4단계 각 3초. ✓ 완료, ● 현재, ○ 대기.

에센스 한 줄 32px+ 세리프. 6축은 텍스트 리스트. 팔레트는 사진 추출 5색 원형.
공유 버튼은 이 화면에 없음 — 대화 후 발견이 생긴 뒤 프로필 화면에서 공유.

#### 에센스 → 채팅 전환 (BUG-003 수정)

```
사용자가 답 입력 + [답하기] 탭
  ↓
즉시 /chat 으로 전환 (fade 300ms)
  ↓
채팅 화면에 이미 표시됨:
  [AI] 에센스 요약 + 첫 질문
  [사용자] 방금 입력한 답
  [AI 타이핑 중...]
  ↓
두 번째 턴 AI 응답 도착 → 바로 표시
  ↓
이후 일반 채팅처럼 자유롭게 주고받기 (추가 전송 필요 없음)
```

- `[답하기]` 버튼: 텍스트 비어있으면 비활성(회색), 입력 있으면 활성
- 전환 후 입력 영역 즉시 활성화 — 추가 메시지 바로 보낼 수 있어야 함
- 에센스 결과 화면의 응답 = 채팅의 첫 번째 사용자 메시지로 세션에 저장

## `/chat` — 메타대화

```
┌──────────────────────────────────────┐
│  ← 홈                        프로필  │
│                                      │
│   ┌─────────────────────────┐        │
│   │ AI: 왜 풍경만 올렸어?    │        │
│   └─────────────────────────┘        │
│            ┌─────────────────────┐   │
│            │ 나: 색감이 좋아서요  │   │
│            └─────────────────────┘   │
│   ┌─────────────────────────┐        │
│   │ AI: 어떤 색감? 차갑고    │        │
│   │ 넓은 느낌? 따뜻한 빛?   │        │
│   └─────────────────────────┘        │
│   ┌─────────────────────────────┐    │
│   │ 발견: "풍경을 좋아하는 게   │    │
│   │  아니라 넓은 곳이 필요한    │    │
│   │  상태구나"                  │    │
│   │              [ 저장할까요? ] │    │
│   │  ┄ 에센스: 양감 · 온도 ┄   │    │
│   └─────────────────────────────┘    │
│   ┌────────────────────────┐         │
│   │ 메시지 입력           →  │         │
│   └────────────────────────┘         │
└──────────────────────────────────────┘
```

#### 채팅 레이아웃 — 고정 입력, 스크롤 메시지

**핵심: 새 메시지가 아래에 추가되고, 자동 스크롤. 채팅이 위로 밀려 올라가지 않는다.**

```
┌──────────────────────────────────────┐
│  ← 홈                                │  ← 헤더 (고정)
│  ┌─── 스크롤 영역 ──────────────┐    │
│  │  [AI 메시지]                  │    │
│  │       [사용자 메시지]          │    │
│  │  [AI 메시지]                  │    │
│  │  [인사이트 카드]               │    │  ← 최신 메시지가 항상 보임
│  │  [AI 타이핑 중...]            │    │
│  └───────────────────────────────┘    │
│  ┌────────────────────────────┐      │  ← 입력 영역 (키보드 위 고정)
│  │ 메시지 입력           📷 [→] │      │
│  └────────────────────────────┘      │
└──────────────────────────────────────┘
```

**채팅 중에는 하단 탭바를 숨긴다.** 나가려면 헤더 `← 홈` 사용.

#### 스크롤 규칙

| 상황 | 동작 |
|------|------|
| 새 AI 메시지 도착 (스크롤이 맨 아래) | 자동 스크롤하여 새 메시지 표시 |
| 사용자가 위로 스크롤한 상태에서 새 메시지 | 자동 스크롤 안 함. "↓ 새 메시지" 플로팅 버튼 |
| 사용자 메시지 전송 | 항상 맨 아래로 스크롤 |
| 키보드 올라올 때 | 마지막 메시지가 보이도록 스크롤 조정 |

"맨 아래" = `scrollHeight - scrollTop - clientHeight < 100px`

#### 사진 전송 후 상태 리셋 (BUG-004 수정)

```typescript
const sendMessage = async (content, type) => {
  try {
    setIsSending(true);
    // API 호출
  } catch (error) {
    // 에러 처리
  } finally {
    setIsSending(false);      // 반드시 리셋
    setAttachedPhotos([]);     // 첨부 사진 클리어
    inputRef.current?.focus(); // 입력 필드 포커스
  }
};
```

- 사진이든 텍스트든 전송 후 동일하게 상태 리셋
- AI 응답 대기 중: 입력은 가능하지만 전송은 불가 (`[→]` 비활성)
- AI 응답 도착 후: `[→]` 즉시 활성화

#### 세션 유지 (BUG-007 수정)

| 시나리오 | 동작 |
|---------|------|
| 채팅 중 헤더 `← 홈` 탭 | `/home`으로 이동. 세션 저장됨 |
| `/home`에서 다시 채팅 시작 | 24시간 이내: "이어서 이야기할까요?" 배너 표시 |
| 브라우저 닫기 → 재방문 | 마지막 세션 24시간 이내면 이어서 가능 |

채팅 중 탭바가 숨겨지므로 "탭 이동 후 세션 사라짐" 문제 자체가 발생하지 않음.

#### 입력 종류별 AI 첫 반응

| 입력 | 첫 반응 | 탐색 방향 |
|------|---------|----------|
| 풍경 사진 | "왜 풍경만 올렸어?" | 색감/구도/기억 → 현재 상태 연결 |
| 글 링크 | "이 글에서 뭐가 걸렸어?" | 역추적 → 에센스 연결 → 발견 |
| 텍스트 메모 | "이 생각이 왜 지금 떠올랐어?" | 맥락 → 패턴 감지 → 발견 |
| 사물 사진 | "이걸 왜 찍었어?" | 감각 축 분석 → 에센스 연결 |

#### 채팅 엣지케이스

| 케이스 | 처리 |
|--------|------|
| 빈 메시지 전송 | `[→]` 비활성이므로 불가능 |
| 2000자 초과 | 제한 + 카운터 표시 |
| AI 응답 파싱 실패 | "다시 생각해볼게요..." → 자동 재시도 1회 |
| 짧은 응답 ("응", "ㅇㅇ") | AI가 구체적 질문: "좀 더 이야기해줄 수 있어요?" |
| 네트워크 끊김 중 전송 | 로컬 큐 저장 + 빨간 `!` 표시. 탭 시 재전송 |
