# 랄프톤 — 에러 핸들링

---

## 에러 케이스 6개

### 1. 유사 사진 과다

**조건**: 사진 여러 장 입력 시 3장 이상이 비슷한 경우

**메시지**:
> 비슷한 사진이 여러 장이에요. 다른 상황의 사진도 넣어보면 더 풍부하게 읽을 수 있어요.

**동작**: 분석은 진행하되, 안내 토스트 표시. 사용자가 교체할 수도, 무시할 수도 있다.

---

### 2. URL 접근 불가

**조건**: 글 링크 입력 시 URL을 열 수 없는 경우

**메시지**:
> 이 링크를 열 수 없어요. 글의 핵심 부분을 직접 붙여넣어볼래요?

**동작**: 텍스트 입력으로 전환 유도. 대화는 중단되지 않는다.

---

### 3. Claude API 파싱 실패

**조건**: API 응답이 JSON 파싱에 실패한 경우

**메시지**:
> 다시 읽어볼게요...

**동작**: 자동 재시도 1회. 재시도도 실패하면 "잠시 후 다시 시도해주세요" 표시.

---

### 4. 텍스트 스크린샷 업로드

**조건**: 사진으로 올렸는데 내용이 텍스트 스크린샷인 경우

**메시지**:
> 이건 글이네요! 텍스트로 넣으면 더 잘 읽을 수 있어요.

**동작**: 텍스트 입력으로 전환 제안. 사진 분석도 가능하지만 텍스트가 더 정확하다는 안내.

---

### 5. 이미지 업로드 실패

**조건**: 네트워크 오류 등으로 이미지 업로드가 실패한 경우

**메시지**:
> 사진을 올리지 못했어요. 다시 시도해볼게요.

**동작**: 자동 재시도 1회. 실패 시 수동 재시도 버튼 표시.

---

### 6. 대화 중 네트워크 끊김

**조건**: 메타대화 진행 중 네트워크가 끊긴 경우

**메시지**: (재연결 후) 이전 대화 이어서 진행.

**동작**:
- 전송 실패한 메시지를 로컬 큐에 임시 저장
- 재연결 시 자동 전송
- 대화 히스토리는 Supabase에 있으므로 유실 없음

---

### 7. Supabase 네트워크 오류

**조건**: Supabase API 호출 시 네트워크 오류 또는 타임아웃 (5초+)

**메시지**:
> 저장하는 데 잠깐 문제가 있었어요. 다시 시도할게요.

**동작**: 자동 재시도 2회 (1초, 3초 간격). 재시도 실패 시 로컬 큐에 저장하고 백그라운드에서 재시도. 사용자에게는 대화를 계속 진행할 수 있게 한다.

---

### 8. Supabase RLS 권한 오류

**조건**: RLS 정책으로 인해 다른 사용자 데이터 접근 차단

**메시지**: (사용자에게 노출하지 않음)

**동작**: 콘솔 에러 로깅. 현재 사용자 세션 확인 후 자동 복구 시도. 복구 불가 시 텔레그램 uid 재인증.

---

### 9. Supabase 연결 초기화 실패

**조건**: 앱 로드 시 Supabase 클라이언트 초기화 실패 (환경변수 누락 등)

**메시지**:
> 서비스에 연결할 수 없어요. 잠시 후 다시 시도해주세요.

**동작**: 환경변수 확인 로그 출력. 재시도 버튼 표시. 3회 실패 시 관리자 알림.
