# 랄프톤 — 제품 정의

> 뭐든 넣으면 당신이 보입니다.

AI가 사진, 글, 메모에서 취향 에센스를 읽고, 호기심 질문으로 자기 이해를 깊게 만드는 서비스.

---

## 핵심 가설

- 사람들은 자기가 뭘 좋아하는지는 알지만, 왜 좋아하는지는 모른다
- 끌리는 사진들의 교집합을 자연어로 읽고 태그로 정리하면 "이게 나야?" 순간이 온다. 뭐든 drop하면 내용과 무드를 같이 읽고 에센스 한 줄로 돌려준다
- 그 언어가 메타대화의 자연스러운 입구가 된다
- 뭐든 넣으면 바로 호기심 질문이 시작되어야 한다 — 메타대화는 별도 단계가 아니라 모든 입력의 즉시 반응
- 축적될수록 떠날 수 없다

---

## 채널 아키텍처

| 채널 | 역할 | 특징 |
|------|------|------|
| **텔레그램** | 대화 시작점 | 사진/글/메모를 보내면 바로 메타대화 시작. 가벼운 인터페이스 |
| **웹앱** | 축적이 잘 보이는 곳 | 에센스, 주제 모음, 프로필 |

어디서 시작하든 같은 MetaProfile에 쌓인다.

텔레그램은 입력이 쉽고, 웹앱은 보기가 쉽다. 입력은 텔레그램에서 하고, 축적된 결과는 웹에서 본다 — 이렇게 나뉘는 게 자연스럽다.

### 하단 탭 네비게이션

| 탭 | 경로 | 표시 조건 |
|----|------|----------|
| 홈 | `/home` | 항상 |
| 세션 | `/sessions` | 항상 |
| 프로필 | `/profile` | 항상 |

랜딩(`/`)과 읽기 결과(`/result`)에서는 하단 탭을 숨긴다.

---

## 페이지 구조

| 페이지 | 경로 | 역할 |
|--------|------|------|
| 랜딩+온보딩 | `/` | 풀스크린. "뭐든 넣으면 당신이 보입니다" + drop 버튼 + 예시 카드 |
| 읽기 결과 | `/result` | drop 후 AI 읽기 결과. 대화 진입은 선택 (스킵 가능) |
| 홈 | `/home` | 일상 진입점. drop 입력 + 최근 읽기 프리뷰 + 주제 모음 칩 |
| 세션 | `/sessions` | 전체 읽기 목록. 매거진 그리드 (사진/날짜 + 인사이트 + 태그) |
| 대화 | `/chat` | 읽기 결과에 반응했을 때 열리는 메타대화 |
| 대화 상세 | `/chat/:sessionId` | 과거 세션 다시 보기 |
| 주제 상세 | `/themes` | 주제별 폴더 카드 스택. 탭하면 해당 주제 읽기 목록 |
| 프로필 | `/profile` | 에센스 + 이번 달 흐름 + 발견 + 태그 |

## 성공 기준

| 기준 | 확인 방법 |
|------|----------|
| 에센스 결과에서 "이게 나야?" 반응 | 시연 중 심사위원/관객 표정, 반응 |
| 에센스 분석 일관성 | 같은 사진 3회 돌려서 방향이 비슷한지 |
| 에센스 → 대화 전환이 자연스러운가 | 열린 질문에 자유 응답 후 대화 진입까지 끊김 없는지 |
| 뭐든 넣으면 즉시 대화 시작 | 사진/글/메모 각각 테스트, 3초 내 첫 질문 |
| 메타대화가 요약이 아니라 탐색 | 질문 10개 중 요약 0개 |
| 에센스가 결론이 아니라 시작점으로 느껴지는가 | "이렇게 읽어봤어요" 톤이 유지되는지 |
| 3분 안에 첫 결과 도달 | 랜딩 → 업로드 → 에센스까지 소요 시간 |
| 주제 클러스터 직관성 | 주제별로 묶인 읽기를 설명 없이 이해하는지 |
| 발견 후 공유 의지 | 대화에서 발견이 생긴 뒤 프로필에서 공유 의지 |

## 리스크

| 리스크 | 대응 |
|--------|------|
| 에센스 분석이 매번 달라짐 | few-shot 예시를 프롬프트에 포함 + temperature 낮추기 |
| "끌리는 사진"이 아니라 일상 기록을 올림 | 업로드 안내 카피로 유도 |
| 에센스가 뻔한 결과 | 가설 질문이 구체적이어야 함. 틀려도 되지만 찔러야 한다 |
| Claude Vision API 속도 (10초+) | 이미지 리사이즈 + 4단계 로딩 애니메이션 |
| 메타대화가 요약 모드로 빠짐 | "요약하지 마라" 명시 + 입력별 첫 질문 고정 |
| 호기심만으로 진입이 안 됨 | 랜딩 카피 A/B 테스트 |
| 주제 클러스터링 품질 | 초기 태그 기반, 20개+ 쌓이면 의미 유사도 전환 |
| 공유 이미지 퀄리티 | OG 이미지 템플릿 사전 제작 |
| 입력 종류 다양 → 대화 품질 편차 | 입력 종류별 프롬프트 분기 + few-shot 각각 준비 |
