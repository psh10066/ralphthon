# 랄프톤 — 인터랙션 상세 스펙

> 모든 화면의 모든 버튼, 모든 상태, 모든 엣지케이스.
> "이걸 읽으면 개발자가 질문 없이 만들 수 있어야 한다."

---

## 용어 정의

| 용어 | 의미 |
|------|------|
| 탭 | 모바일 터치 (클릭) |
| 길게 누르기 | 500ms 이상 터치 유지 |
| 스와이프 | 좌우 또는 상하 드래그 제스처 |
| 바텀시트 | 하단에서 올라오는 반투명 모달 |
| 토스트 | 하단에 2초간 표시되는 알림 메시지 |
| safe area | 노치/다이나믹 아일랜드/하단 인디케이터 영역 제외한 안전 영역 |

## 0. 글로벌 규칙

### 0-1. 네비게이션

```
[하단 탭바]
───  ───  ───
 홈   세션  프로필
```

- 하단 탭바는 **항상** 표시. 예외: 온보딩 퍼널(`/`), 채팅 중(`/chat`)
- 채팅 중에는 탭바 대신 입력 영역이 하단을 차지
- 현재 탭은 `#3D3632` 진하게, 비활성 탭은 `#C4C0BC`
- 탭 전환 시 페이드 트랜지션 200ms
- 각 탭은 독립 스크롤 위치를 기억 (탭 전환 후 돌아오면 원래 위치)

### 0-2. 뒤로가기

| 화면 | iOS 스와이프백 | 안드로이드 백버튼 | 헤더 ← |
|------|-------------|----------------|--------|
| 온보딩 퍼널 | 이전 섹션 | 이전 섹션 | 없음 |
| `/chat` | `/home` | `/home` | `/home` |
| `/chat/:id` | `/sessions` | `/sessions` | `/sessions` |
| `/profile` | - (탭) | - (탭) | - (탭) |
| `/sessions` | - (탭) | - (탭) | - (탭) |

### 0-3. 로딩 상태

모든 API 호출에 적용:

| 단계 | 시간 | 표시 |
|------|------|------|
| 즉시 | 0-300ms | 아무것도 안 보여줌 (깜빡임 방지) |
| 대기 | 300ms-3s | 스켈레톤 또는 스피너 |
| 지연 | 3s-10s | "조금만 기다려주세요..." 텍스트 추가 |
| 타임아웃 | 10s+ | "다시 시도할게요" + 자동 재시도 1회 |
| 실패 | 재시도도 실패 | "연결이 불안정해요. 잠시 후 다시 시도해주세요" + 수동 재시도 버튼 |

### 0-4. 빈 상태 (Empty State)

| 화면 | 조건 | 표시 |
|------|------|------|
| `/home` 최근 발견 | 대화 0회 | "아직 발견이 없어요. 뭐든 가져와볼래요?" |
| `/home` 주제 모음 | 세션 < 3 | 주제 모음 영역 숨김 |
| `/sessions` | 대화 0회 | "아직 대화한 적 없어요. 홈에서 시작해볼래요?" + 홈 탭 이동 버튼 |
| `/profile` 발견 목록 | 발견 0개 | "대화에서 발견이 쌓이면 여기 나타나요" |
| `/profile` 주제 모음 | 세션 < 5 | "아직 주제가 모이는 중이에요" + 스케치 플레이스홀더 |
| `/profile` 반복 키워드 | 키워드 < 3 | 섹션 자체를 숨김 |

### 0-5. 네트워크

| 상태 | 감지 | 표시 | 동작 |
|------|------|------|------|
| 오프라인 | `navigator.onLine === false` | 상단 바: "연결이 끊겼어요" (빨간 배경 아님, `#F5F3F0` + 텍스트) | 입력은 로컬 큐에 저장. 전송 버튼 비활성 |
| 재연결 | `online` 이벤트 | "다시 연결됐어요" 2초 후 사라짐 | 큐에 쌓인 메시지 자동 전송 |
| 느린 연결 | API 응답 > 5s | 일반 로딩 상태 유지 (별도 알림 없음) | 타임아웃은 15s |

### 0-6. 인증

| 진입점 | 인증 방식 | 세션 유지 |
|--------|---------|----------|
| 텔레그램 봇 | Telegram uid 자동 | 영구 (봇 차단 전까지) |
| 웹앱 (텔레그램 내) | Telegram WebApp initData | 텔레그램 앱 켜져 있는 동안 |
| 웹앱 (브라우저 직접) | 텔레그램 로그인 위젯 | 30일 (Supabase auth) |
| 첫 방문 (미인증) | 없음 | 온보딩 퍼널은 인증 없이 가능. 에센스 결과 저장 시 인증 필요 |

미인증 → 에센스 분석까지는 가능. 결과 저장 + 대화 시작 시 텔레그램 로그인 요구.
로그인 후 분석 결과를 계정에 연결.

### 0-7. 모바일 키보드

- 키보드가 올라오면 **채팅 입력 영역은 키보드 위에 고정**
- 채팅 메시지 영역은 키보드 높이만큼 줄어듦 (최신 메시지가 보이도록)
- `visualViewport` API로 키보드 높이 감지
- iOS Safari: `position: fixed` + `bottom: env(keyboard-inset-height)` 패턴
- 키보드 올라올 때 현재 스크롤 위치 유지 (점프 금지)

## 1. 온보딩 퍼널 (`/`)

### 1-1. 랜딩 (뷰포트 1)

**진입 조건**: 첫 방문 또는 에센스 미생성 사용자
**재방문 사용자**: 에센스 있으면 `/home`으로 리다이렉트

#### 인터랙션

| 요소 | 동작 | 상세 |
|------|------|------|
| `[ 시작하기 ]` 버튼 | 탭 → 업로드 섹션으로 스크롤 | smooth scroll, 500ms |
| 예시 에센스 카드 (3개) | 좌우 스와이프 | `scroll-snap-type: x mandatory` |
| 예시 카드 탭 | 카드 상세 바텀시트 | 읽기 결과 + 태그 표시 |
| 바텀시트 내 `[ 나도 해볼래 ]` | 바텀시트 닫기 + 업로드로 스크롤 | |
| 바텀시트 바깥 탭 | 바텀시트 닫기 | |
| 아래로 스크롤 | 업로드 섹션 도달 | 스크롤 자유 |

#### 엣지케이스

| 케이스 | 처리 |
|--------|------|
| 이미 에센스 있는 사용자가 `/` 직접 접근 | `/home`으로 302 리다이렉트 |
| 예시 카드 로딩 실패 | 카드 영역 숨김. 랜딩 카피 + 버튼만 표시 |
| 초고속 스크롤로 업로드 건너뜀 | 업로드가 퍼널의 마지막이므로 건너뛸 곳 없음 |

### 1-2. 업로드 (뷰포트 2)

**핵심 변경: 하나씩 클릭이 아니라 한꺼번에 선택**

#### 업로드 방식

| 방식 | 트리거 | 동작 |
|------|--------|------|
| **갤러리 멀티셀렉트** | `[ + 사진 고르기 ]` 버튼 탭 | `<input type="file" multiple accept="image/*">` → 네이티브 갤러리에서 최대 5장 선택 |
| **드래그 앤 드롭** | 데스크톱에서 파일 드래그 | 드롭존 하이라이트 (`border: 2px dashed #C4C0BC`) |
| **카메라 촬영** | `[ 📷 촬영 ]` 버튼 탭 | `capture="environment"` → 카메라 직접 실행 → 1장씩 추가 |
| **클립보드 붙여넣기** | Ctrl+V / 길게 누르기 → 붙여넣기 | 이미지 감지 시 자동 추가 |

#### 업로드 영역 상태

| 상태 | 표시 | CTA |
|------|------|-----|
| 입력 없음 | `[ + 뭐든 drop ]` 큰 버튼 (전체 영역) + "끌리는 거 뭐든. 사진, 글, 링크." | 없음 |
| 입력 있음 | drop한 항목 미리보기 | `[ 읽기 시작 ]` **활성** (진한색, 탭 가능) |
| 업로드 중 | 항목에 프로그레스 오버레이 (반투명 + 퍼센트) | CTA 비활성 |
| 업로드 중 | 각 사진에 프로그레스 오버레이 (반투명 + 퍼센트) | CTA 비활성 |

#### 입력 미리보기 레이아웃

- drop한 항목(사진/글/링크)을 단일 프리뷰 카드로 표시
- 사진의 경우 `object-fit: cover` 정사각 크롭
- 글/링크의 경우 텍스트 미리보기 카드

#### 개별 사진 인터랙션

| 요소 | 동작 | 상세 |
|------|------|------|
| 사진 썸네일 탭 | 풀스크린 프리뷰 | 핀치 줌 가능. 상단에 `[ × ]` 닫기 |
| 사진 우상단 `[ × ]` | 해당 사진 제거 | 나머지 사진 앞으로 당김. "N/5" 업데이트 |
| 사진 길게 누르기 | 드래그로 순서 변경 | 살짝 들어올리는 애니메이션 + 햅틱 |
| `[ + ]` 빈 슬롯 탭 | 갤러리 열기 (추가 선택) | 남은 장수만큼만 선택 가능 |

#### 이미지 처리 파이프라인

```
사용자가 사진 선택
  ↓
클라이언트 리사이즈 (max 1920px, JPEG quality 0.8)
  ↓
용량 체크: 5MB 이하 → 통과 / 초과 → 재압축 (quality 0.6)
  ↓
재압축 후에도 5MB 초과 → 토스트: "이 사진은 너무 커서 줄여볼게요" → max 1280px로 추가 리사이즈
  ↓
그래도 실패 → 토스트: "이 사진은 올릴 수 없어요. 다른 사진을 골라볼래요?" + 해당 슬롯 비움
  ↓
base64 변환 → 메모리에 저장 (Supabase 업로드는 분석 시점)
```

**핵심: 사용자는 리사이즈/압축을 인지하지 못한다. 모든 처리는 자동.**

#### 엣지케이스

| 케이스 | 처리 |
|--------|------|
| HEIC/HEIF (iPhone 기본) | 클라이언트에서 JPEG 변환 (`heic2any` 라이브러리) |
| GIF 업로드 | 첫 프레임만 추출하여 정지 이미지로 |
| 동영상 파일 선택 | 무시 + 토스트: "사진만 올릴 수 있어요" |
| 같은 사진 중복 | 허용 (사용자 의도일 수 있음. 유사 사진 경고는 분석 시점에) |
| 입력 없이 `[ 읽기 시작 ]` 탭 | 버튼 자체가 비활성 — 탭 이벤트 무시. 시각적으로 회색 |
| 항목 drop 후 제거 | CTA 즉시 비활성화 |
| 갤러리 권한 거부 | 토스트: "사진에 접근할 수 없어요. 설정에서 허용해주세요" + 설정 열기 딥링크 (가능한 경우) |
| 네트워크 없이 업로드 | base64는 로컬 메모리에 저장되므로 오프라인에서도 사진 선택 가능. 분석 버튼만 비활성 + "인터넷이 필요해요" |
| 브라우저 메모리 부족 | 고해상도 이미지 drop 시 저사양 기기에서 문제 가능 → 리사이즈를 먼저 하고 원본은 즉시 해제 |
| 페이지 새로고침 | 선택한 사진 유실. 복구 불가. (localStorage에 base64 저장은 용량 문제) |
| 업로드 중 뒤로가기 | "사진을 선택하고 있었어요. 나갈까요?" 확인 다이얼로그 없음 — 그냥 나감 (선택 유실) |

### 1-3. 로딩 → 에센스

`[ 읽기 시작 ]` 탭 후:

#### 전체 흐름

```
[읽기 시작] 탭
  ↓
로딩 화면 전환 (fade 300ms)
  ↓
입력 항목 base64 → POST /api/analyze 호출
  ↓
4단계 로딩 애니메이션 (타이머 기반, API 응답과 독립)
  ↓
API 응답 도착 → 4단계 모두 완료 처리 → 0.5초 대기
  ↓
에센스 결과 화면 전환 (fade 500ms)
```

#### 로딩 타이밍 규칙

| 상황 | 동작 |
|------|------|
| API가 12초 내 응답 | 타이머가 4단계까지 진행. 이미 3단계인데 응답 오면 → 4단계 빠르게 완료 → 결과 표시 |
| API가 12초 넘김 | 4단계 "언어로 옮기고 있어요"에서 대기. 펄스 애니메이션 계속 |
| API가 30초 넘김 | "시간이 좀 걸리고 있어요..." 텍스트 추가. 대기 유지 |
| API가 60초 넘김 (타임아웃) | "다시 시도할게요" → 자동 재시도 1회 |
| 재시도도 실패 | "지금 읽기가 어려워요. 잠시 후 다시 시도해주세요" + `[ 다시 시도 ]` 버튼. 업로드 화면으로 돌아가지 않음 (사진 유지) |

#### 로딩 중 인터랙션

| 동작 | 처리 |
|------|------|
| 뒤로가기 / 브라우저 백 | 분석 취소하지 않음. "분석 중이에요. 결과가 나오면 알려줄게요" 토스트 → 홈으로 이동 (분석 완료 시 push notification 또는 재방문 시 결과 표시) |
| 화면 터치 | 아무 반응 없음 |
| 앱 백그라운드 → 포그라운드 | 분석 상태 체크. 완료됐으면 결과 표시. 아니면 현재 단계 유지 |

### 1-4. 읽기 결과 — 에센스 결과 + 첫 대화 전환

#### 화면 구성 (위에서 아래로 스크롤)

```
┌──────────────────────────────────────┐
│   [drop한 항목 미리보기]               │  ← 스크롤 맨 위
│                                      │
│   사진에서 이렇게 읽어봤어요.          │
│                                      │
│   "손 닿는 곳의 아늑함을              │  ← 에센스 한 줄 (세리프, 가장 큼)
│    모으는 사람"                        │
│                                      │
│   #공간 #고요한  (주제·스타일 태그)               │
│                                      │
│   ● ● ● ● ●  (컬러 팔레트)          │
│                                      │
│   ─── 구분선 ───                     │
│                                      │
│   이게 다는 아닐 수 있어요.            │  ← 관찰 + 질문 영역
│   더 알고 싶어서 하나 물어볼게요.      │
│                                      │
│   "{observation}"                    │
│   {firstQuestion}                    │
│                                      │
│   ┌────────────────────────────┐     │  ← 입력 필드 (여기가 핵심)
│   │ 여기에 자유롭게 답해주세요   │     │
│   └────────────────────────────┘     │
│              [ 답하기 ]               │
│                                      │
└──────────────────────────────────────┘
```

#### 인터랙션 상세

| 요소 | 동작 | 상세 |
|------|------|------|
| 입력 미리보기 | 탭 시 풀스크린 (사진인 경우) 또는 원문 표시 |  |
| 6축 각 행 (보조) | 탭 → 토글 펼침 | "양감이 높다는 건 볼륨감 있는 것에 끌린다는 뜻이에요" 부연 |
| 태그 칩 | 탭 → 관련 읽기 목록 |  |
| 팔레트 각 원 | 탭 → 해당 색이 추출된 원본 사진 하이라이트 | 사진 스트립의 해당 사진에 테두리 |
| 입력 필드 | 탭 → 키보드 올라옴 → 자동 스크롤하여 입력 필드가 키보드 위에 보임 | |
| `[ 답하기 ]` 버튼 | 텍스트 비어있으면 비활성. 입력 있으면 활성 | 활성: `#3D3632`, 비활성: `#C4C0BC` |

#### `/chat` 전환 상세

```

  ↓

  ↓

  ↓
POST /api/chat 호출 (사용자 응답 + 에센스 컨텍스트)
  ↓
[AI] 두 번째 턴 응답 (가설 포함)
```

#### 엣지케이스

| 케이스 | 처리 |
|--------|------|

| 입력 필드에 500자 초과 | 500자 제한 + 카운터 표시 "432/500" |
| 입력 필드에 URL만 입력 | 허용. 대화 시작 시 link 타입으로 처리 |

| 읽기 결과가 비정상 (태그 누락) | 태그 없으면 빈 배열로 처리 |
| 브라우저 뒤로가기 | 업로드 화면으로 돌아감. 사진은 유실됨 (이미 분석 완료) |
| 같은 사진으로 재분석 요청 | 현재 버전에서는 미지원. `/profile`에서 "다시 읽어보기" 기능으로 제공 예정 |

## 3. 채팅 (`/chat`)

### 3-1. 레이아웃 — 고정 입력, 스크롤 메시지

**핵심: 채팅이 위로 올라가지 않는다. 새 메시지가 아래에 추가되고, 자동 스크롤.**

```
┌──────────────────────────────────────┐
│  ← 홈                                │  ← 헤더 (고정)
│                                      │
│  ┌─── 스크롤 영역 ──────────────┐    │
│  │                               │    │
│  │  [AI 메시지]                  │    │
│  │                               │    │
│  │       [사용자 메시지]          │    │
│  │                               │    │
│  │  [AI 메시지]                  │    │
│  │                               │    │
│  │  [인사이트 카드]               │    │
│  │                               │    │
│  │       [사용자 메시지]          │    │  ← 최신 메시지가 항상 보임
│  │                               │    │
│  │  [AI 타이핑 중...]            │    │
│  │                               │    │
│  └───────────────────────────────┘    │
│                                      │
│  ┌────────────────────────────┐      │  ← 입력 영역 (키보드 위 고정)
│  │ 메시지 입력              📷  │      │
│  │                         [ → ]│      │
│  └────────────────────────────┘      │
└──────────────────────────────────────┘
```

#### 스크롤 동작 규칙

| 상황 | 동작 |
|------|------|
| 새 AI 메시지 도착 | 스크롤이 맨 아래에 있었으면 → 자동 스크롤하여 새 메시지 표시 |
| 사용자가 위로 스크롤한 상태에서 새 메시지 | 자동 스크롤 하지 않음. 하단에 "↓ 새 메시지" 플로팅 버튼 표시 |
| "↓ 새 메시지" 탭 | 맨 아래로 스크롤 (smooth) |
| 사용자 메시지 전송 | 항상 맨 아래로 스크롤 |
| 키보드 올라올 때 | 현재 보이는 마지막 메시지가 계속 보이도록 스크롤 조정 |

#### 스크롤 위치 판정

```typescript
// "맨 아래"의 정의: 스크롤 바닥에서 100px 이내
const isAtBottom = scrollArea.scrollHeight - scrollArea.scrollTop - scrollArea.clientHeight < 100;
```

### 3-2. 메시지 타입별 렌더링

#### AI 메시지 (왼쪽)

```
┌─────────────────────────────┐
│  텍스트 내용                 │  ← 배경 없음 (transparent)
│                             │     왼쪽 정렬
│                             │     Pretendard 15px/300
│                             │     max-width: 85%
└─────────────────────────────┘
```

#### 사용자 메시지 (오른쪽)

```
                  ┌─────────────────────┐
                  │  텍스트 내용         │  ← #F5F3F0 배경
                  │                     │     오른쪽 정렬
                  │                     │     border-radius: 16px
                  └─────────────────────┘
```

#### 사용자 사진 메시지

```
                  ┌─────────────────────┐
                  │ ┌─────┐ ┌─────┐    │
                  │ │ 📷  │ │ 📷  │    │  ← 썸네일 max 280px 너비
                  │ └─────┘ └─────┘    │     탭 시 풀스크린
                  │ (텍스트가 있으면)    │
                  └─────────────────────┘
```

#### 인사이트 카드 (AI가 발견 감지 시)

```
┌─────────────────────────────────┐
│  💡 발견                        │  ← 왼쪽에 팔레트 primary 세로선 2px
│                                 │
│  "나는 만드는 건 잘하는데        │
│   닫는 게 안 돼"                │
│                                 │
│  #Closer  #통제                 │
│                                 │

└─────────────────────────────────┘
```

| 버튼 | 동작 |
|------|------|
| `[ 기록할까요? ]` | 인사이트 저장 (Supabase) → 카드가 "저장됨 ✓"로 변경 → 대화 계속 |
| `[ 넘어가기 ]` | 카드 페이드아웃 → 대화 계속 |
| 두 버튼 다 안 누르고 대화 계속 | 카드는 그대로 유지. 나중에 스크롤해서 저장 가능 |

```

│  · · ·          │  ← 점 3개 순차 페이드 (0.8초 주기)
└─────────────────┘
```

- API 호출 시작 즉시 표시
- API 응답 도착하면 타이핑 인디케이터 → AI 메시지로 교체 (200ms fade)

### 3-3. 입력 영역 상세

| 상태 | 표시 |
|------|------|
| 비어있음 | 플레이스홀더: "무슨 생각이에요?" (회색) |
| 텍스트 입력 중 | 자동 높이 확장 (1줄 → 최대 5줄). 5줄 초과 시 내부 스크롤 |
| 사진 첨부 | 입력 영역 위에 썸네일 행 (최대 3장) |
| 전송 중 | `[ → ]` → 스피너. 입력 불가 |
| AI 응답 대기 중 | 입력 가능 (하지만 전송 불가). `[ → ]` 비활성 |

**전송 규칙**:
- `[ → ]` 버튼으로만 전송
- Enter = 줄바꿈 (전송 아님)
- 전송 후 입력 필드 비움
- AI가 응답 중일 때는 전송 불가 (큐잉 없음)

### 3-4. 세션 관리

| 시나리오 | 동작 |
|---------|------|

| 24시간 초과 | 새 세션으로 시작 |
| 대화 중 다른 탭 이동 → 돌아옴 | 같은 세션 유지 |
| 브라우저 닫기 → 재방문 | 마지막 세션이 24시간 이내면 이어서, 아니면 새 세션 |

### 3-5. 대화 종료

명시적 종료 버튼 없음. 자연스럽게 끝남.

|------|---------|

| 사용자가 뒤로가기 | `/home`으로 이동. 세션은 저장됨 |
| 10턴 이상 반복 패턴 | AI가 자연스럽게 정리: "오늘 대화에서 이런 게 보였어요" → 인사이트 카드 |

### 3-6. 엣지케이스

| 케이스 | 처리 |
|--------|------|
| 빈 메시지 전송 시도 | `[ → ]` 비활성이므로 불가능 |

| 욕설/부정적 표현 | 안전장치 발동. 패턴 감지 후 "다른 각도에서 볼까요?" |

| 대화 버블 | 읽기 전용. 길게 누르기 → 복사 |
| 인사이트 카드 | 탭 → 상세 펼침 |
| "이 대화 이어하기" 버튼 (하단) | `/chat`으로 전환. 해당 세션 이어서 (24시간 규칙 무시) |
| 스크롤 | 자유 |

### 4-3. 엣지케이스

| 케이스 | 처리 |
|--------|------|
| 존재하지 않는 sessionId | 404 페이지: "이 대화를 찾을 수 없어요" + `/sessions`로 이동 버튼 |
| 다른 사용자의 세션 접근 | RLS로 차단. 404와 동일한 화면 (정보 노출 방지) |
| 매우 긴 대화 (50턴+) | 가상 스크롤 (windowing). 처음 20턴 로드 → 스크롤 시 추가 로드 |

## 7. 메타대화 엔진 개선 사항

> 현재 문제: "질문만 하고 분석은 안 한다"
> 해결: 메타다이얼로그 코어 로직을 엔진에 심는다

### 7-1. 대화 턴별 전략

| 턴 | 전략 | 예시 |
|----|------|------|
| 1턴 (첫 질문) | 관찰 사실 + 열린 질문. 가설 없이 | "이 사진에서 뭐가 끌려?" |
| 2턴 | 사용자 응답 기반 가설 제시 | "따뜻한 톤에 끌린다면, 차가운 곳에서는 불편해요?" |
| 3-5턴 | 깊이 파기. "왜?" → "그래서 뭘 알 수 있는데?" | "색감이 중요하다면... 옷도 색으로 고르는 편이에요?" |
| 6턴+ | 연결 + 패턴 감지. 에센스 연결은 여기서부터 | "에센스에서 봤던 '채워진 볼륨'이랑 닿는 것 같아요" |
| 발견 감지 | 사용자 말 그대로 정리 + 저장 제안 | 인사이트 카드 표시 |
| 8-10턴 | 자연스러운 정리 또는 새 주제 전환 | "오늘 이런 게 보였어요. 다음에 또 가져와요" |

### 7-2. "분석 안 함" 문제 해결

현재 문제: AI가 안전한 확인 질문만 반복 ("그래요?", "더 이야기해줄래요?")

수정 방향:
1. **가설을 먼저 던지고 틀려도 됨** — "혹시 이런 건 아닌가요?"
2. **연결을 적극적으로 만듦** — 이전 대화, 에센스, 다른 입력 사이의 패턴
3. **요약이 아니라 재프레이밍** — 사용자가 한 말을 다른 각도로 되돌려줌
4. **명시적 발견 선언** — "이거 발견인 것 같아요" + 인사이트 카드

### 7-3. 질문 금지 패턴

| 금지 | 대안 |
|------|------|
| "그래요?" | 가설: "혹시 이런 건 아닌가요?" |
| "더 이야기해줄래요?" | 구체적: "그때 어떤 느낌이었어요?" |
| "어떻게 생각해요?" | 도발적: "근데 이건 반대로 볼 수도 있지 않아요?" |
| "좋은 관점이네요" | 연결: "전에 이런 얘기도 했는데, 닿는 것 같아요" |
| 연속 2회 이상 질문만 | 반드시 관찰/가설/연결 중 하나를 섞어서 |

## 8. 텔레그램 봇 인터랙션

### 8-1. 첫 진입

```
사용자가 봇 /start
  ↓
"안녕하세요. 사진이나 글을 보내주면, 거기서 당신에 대해 읽어볼게요."
  ↓
(사용자가 뭔가 보냄)
  ↓
에센스 미생성: "먼저 에센스를 읽어볼게요. 끌리는 걸 뭐든 하나 보내주세요."
에센스 생성 완료: 바로 메타대화 시작
```

### 8-2. 사진 전송

| 시나리오 | 동작 |
|---------|------|
| 에센스 미생성 + 항목 1개 | 자동으로 에센스 분석 시작 → 결과 텍스트로 전송 → "웹에서 더 자세히 볼 수 있어요" + 웹 링크 |
| 에센스 생성 완료 + 사진 | 메타대화 시작 (photo 타입) |
| 고해상도 사진 (10MB+) | 텔레그램이 자동 압축. 추가 처리 불필요 |

### 8-3. 텍스트/링크 전송

| 시나리오 | 동작 |
|---------|------|
| 일반 텍스트 | memo 타입 메타대화 |
| URL 포함 텍스트 | link 타입 메타대화. URL 크롤링 시도 |
| URL 크롤링 실패 | "이 링크를 열 수 없어요. 글의 핵심 부분을 직접 보내줄래요?" |

### 8-4. 인사이트 감지

```
대화 중 발견 감지
  ↓
인사이트 텍스트 + 태그 전송
  ↓
[웹에서 보기] 인라인 버튼 → 웹앱 `/profile` 열기
```

### 8-5. 엣지케이스

| 케이스 | 처리 |
|--------|------|
| 봇 차단 후 재시작 | 기존 데이터 유지 (uid 기반). /start 시 "돌아오셨네요" |
| 그룹 채팅에서 봇 멘션 | 무시. 1:1 대화만 지원 |
| 스티커/이모지만 보냄 | "사진이나 글을 보내주면 더 잘 읽을 수 있어요" |
| 음성 메시지 | v1에서 미지원. "아직 음성은 읽을 수 없어요. 텍스트로 보내줄래요?" |
| 파일 (PDF 등) | 미지원. "사진이나 텍스트만 읽을 수 있어요" |
| 연속 사진 여러 장 (에센스 완료 후) | 최근 1장만 사용. "가장 최근 사진으로 읽어볼게요" |
