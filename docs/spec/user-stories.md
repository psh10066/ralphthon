# 랄프톤 — 유저스토리

> 모든 기능을 "누가, 뭘, 왜" 기준으로 정의.
> 우선순위: P0 = 없으면 출시 불가, P1 = 첫 주 내 필요, P2 = 이후

---

## 온보딩 퍼널

### US-001: 항목 drop (P0)

**As a** 처음 방문한 사용자
**I want to** 사진, 글, 링크 중 뭐든 하나를 drop하고 싶다
**So that** 무엇을 가져와야 할지 고민하지 않아도 된다

**수락 기준:**
- [ ] `[ + 뭐든 drop ]` 탭 시 입력 방식 선택 (갤러리 / 텍스트 / 링크)
- [ ] 사진 1장 선택 → 미리보기 표시 후 `[ 읽기 시작 ]` 활성
- [ ] 글 입력 → 텍스트 프리뷰 후 `[ 읽기 시작 ]` 활성
- [ ] 링크 붙여넣기 → 링크 프리뷰 카드 후 `[ 읽기 시작 ]` 활성
- [ ] 드래그 앤 드롭(데스크톱)으로도 항목 추가 가능

---

### US-002: 대용량 이미지 자동 처리 (P0)

**As a** iPhone 사용자
**I want to** 고해상도 사진을 올려도 화면이 멈추지 않게 하고 싶다
**So that** 사진을 미리 줄이는 수고 없이 바로 분석할 수 있다

**수락 기준:**
- [ ] 사진 선택 즉시 클라이언트에서 자동 리사이즈 (max 1920px, JPEG 0.8)
- [ ] 5MB 초과 시 추가 압축 (quality 0.6, 필요 시 max 1280px)
- [ ] HEIC/HEIF 포맷 자동 JPEG 변환
- [ ] 리사이즈 과정이 사용자에게 보이지 않는다
- [ ] 1장당 처리 시간 1초 이내
- [ ] 처리 실패 시 토스트: "이 사진은 올릴 수 없어요. 다른 사진을 골라볼래요?"

---

### US-003: 입력 항목 관리 (P0)

**As a** 항목을 drop한 사용자
**I want to** drop한 항목을 제거하고 다시 선택하고 싶다
**So that** 잘못 가져온 항목을 처음부터 다시 하지 않아도 된다

**수락 기준:**
- [ ] 미리보기 우상단 `[ x ]` 탭 시 해당 항목 제거
- [ ] 제거 후 CTA `[ 읽기 시작 ]` 즉시 비활성화
- [ ] 빈 상태로 돌아가면 `[ + 뭐든 drop ]` 다시 표시

---

### US-004: 에센스 분석 로딩 (P0)

**As a** `[ 읽기 시작 ]`을 탭한 사용자
**I want to** 분석이 진행 중이라는 걸 알고 싶다
**So that** 화면이 멈춘 건지 분석 중인지 구분할 수 있다

**수락 기준:**
- [ ] 4단계 로딩 애니메이션 순차 표시 (색감→질감→온도→언어)
- [ ] 각 단계 약 3초, 현재 단계는 펄스 애니메이션
- [ ] API 응답이 12초 내 도착: 남은 단계 빠르게 완료 후 결과 표시
- [ ] API 응답이 30초 초과: "시간이 좀 걸리고 있어요..." 텍스트 추가
- [ ] API 응답이 60초 초과 (타임아웃): 자동 재시도 1회
- [ ] 재시도도 실패: "지금 읽기가 어려워요" + `[ 다시 시도 ]` 버튼

---

### US-005: 에센스 결과 확인 (P0)

**As a** 분석이 완료된 사용자
**I want to** 나의 에센스 결과를 보고 싶다
**So that** AI가 내 사진에서 뭘 읽었는지 알 수 있다

**수락 기준:**
- [ ] 에센스 한 줄이 가장 크게 표시 (세리프, 28px+)
- [ ] 주제 태그 + 스타일 태그 칩 표시
- [ ] 컬러 팔레트 5색 원형 표시
- [ ] "사진에서 이렇게 읽어봤어요" 겸손한 톤
- [ ] drop한 항목 미리보기 상단 표시
- [ ] 6축 각 행 탭 시 부연 설명 토글 (보조 정보)
- [ ] 태그 칩 탭 시 관련 읽기 목록 표시

---

### US-006: 에센스에서 첫 대화 전환 (P0)

**As a** 에센스 결과를 본 사용자
**I want to** 질문에 답하면 바로 대화가 시작되었으면 좋겠다
**So that** 추가 버튼을 또 누르거나 기다리지 않아도 된다

**수락 기준:**
- [ ] 에센스 결과 하단에 관찰 + 열린 질문 + 입력 필드 표시
- [ ] `[ 답하기 ]` 버튼: 입력 비어있으면 비활성(회색), 입력 있으면 활성
- [ ] `[ 답하기 ]` 탭 즉시 `/chat`으로 전환 (300ms fade)
- [ ] 채팅 화면에 AI 첫 턴 + 사용자 응답 + AI 두 번째 턴(자동 호출) 표시
- [ ] 전환 후 입력 영역 즉시 활성화 — 추가 메시지 바로 전송 가능
- [ ] 추가 전송 없이 바로 자유 채팅 가능 (한 번 더 보내야 하는 문제 없음)

## 홈

### US-007: 홈에서 입력하여 대화 시작 (P0)

**As a** 에센스 생성 완료된 사용자
**I want to** 홈에서 사진/글/링크/메모를 입력하면 바로 대화가 시작되었으면 좋겠다
**So that** 뭐든 가져오면 즉시 AI와 이야기할 수 있다

**수락 기준:**
- [ ] 입력 영역에서 텍스트 직접 타이핑 가능 (자동 높이 확장, 최대 5줄)
- [ ] `📷` 탭 → 갤러리 멀티셀렉트 (최대 3장)
- [ ] `🔗` 탭 → URL 입력 또는 클립보드 자동 감지
- [ ] URL 붙여넣기 시 자동 프리뷰 카드 (제목 + 도메인)
- [ ] `[ → ]` 보내기 탭 시 즉시 `/chat`으로 전환
- [ ] 입력 비어있으면 `[ → ]` 비활성

---

### US-008: 최근 발견 확인 (P1)

**As a** 돌아온 사용자
**I want to** 최근 대화에서 나온 발견들을 홈에서 바로 보고 싶다
**So that** 지난 대화의 핵심을 빠르게 떠올릴 수 있다

**수락 기준:**
- [ ] 최근 발견 카드 최대 5개 표시
- [ ] 각 카드: 발견 텍스트 + 날짜 + "N 대화에서"
- [ ] 카드 탭 → 해당 세션 `/chat/:sessionId`로 이동
- [ ] 대화 0회면 이 섹션 숨김

## 채팅

### US-009: 채팅 중 스크롤 자연스러움 (P0)

**As a** AI와 대화 중인 사용자
**I want to** 새 메시지가 오면 자동으로 보이고, 위로 스크롤해도 방해받지 않고 싶다
**So that** 대화에 집중할 수 있다

**수락 기준:**
- [ ] 입력 영역은 화면 하단(키보드 위)에 항상 고정
- [ ] 메시지 영역은 독립 스크롤
- [ ] 새 AI 메시지 도착 시: 맨 아래에 있었으면 자동 스크롤
- [ ] 위로 스크롤한 상태에서 새 메시지: 자동 스크롤 안 함 + "↓ 새 메시지" 버튼
- [ ] 사용자 메시지 전송 시: 항상 맨 아래로 스크롤
- [ ] 키보드 올라올 때: 마지막 메시지가 보이도록 스크롤 조정

---

### US-010: 채팅 중 사진 전송 (P0)

**As a** 대화 중인 사용자
**I want to** 사진을 보내고 나서도 대화를 계속하고 싶다
**So that** 사진 하나 보냈다고 대화가 끊기지 않는다

**수락 기준:**
- [ ] 채팅 입력 영역에서 `📷` 탭 → 갤러리에서 사진 선택 (최대 3장)
- [ ] 선택된 사진 썸네일 프리뷰 표시
- [ ] `[ → ]` 전송 후 AI 응답 도착
- [ ] AI 응답 후 입력 영역 즉시 재활성화 (텍스트/사진 모두 가능)
- [ ] 전송 실패 시에도 입력 영역 재활성화 (finally 블록에서 상태 리셋)

---

### US-011: 채팅 세션 유지 (P0)

**As a** 대화 중인 사용자
**I want to** 다른 화면 갔다 와도 대화가 유지되었으면 좋겠다
**So that** 대화 내용이 사라지지 않는다

**수락 기준:**
- [ ] 채팅 중에는 하단 탭바 숨김 (나가려면 헤더 `← 홈` 사용)
- [ ] `← 홈` 탭 후 다시 채팅 시작: 24시간 이내면 "이어서 이야기할까요?" 배너
- [ ] 배너 탭 → 이전 세션 이어서 대화
- [ ] 브라우저 닫기 → 재방문: 마지막 세션 24시간 이내면 이어서 가능
- [ ] 세션 데이터는 Supabase에 저장 (브라우저 독립)

---

### US-012: AI가 질문만 하지 않고 분석도 함 (P0)

**As a** 대화 중인 사용자
**I want to** AI가 질문만 반복하지 않고 가설도 던지고 발견도 만들어줬으면 좋겠다
**So that** 대화가 루즈하지 않고 실제로 자기 이해가 깊어진다

**수락 기준:**
- [ ] 연속 3턴 이상 질문만 하지 않는다
- [ ] 턴 2에서 사용자 응답 기반 가설을 던진다 (틀려도 됨)
- [ ] 5턴 이상 대화 시 반드시 1회 이상 발견(인사이트) 시도
- [ ] 인사이트 카드: 발견 텍스트 + 태그 + `[ 기록할까요? ]` / `[ 넘어가기 ]`
- [ ] 에센스 연결은 3턴 이후, 세션당 1-2회만
- [ ] 대화 type에 "hypothesis" 포함 (question만이 아님)

---

### US-013: AI가 오타를 지적하지 않음 (P0)

**As a** 편하게 대화하고 싶은 사용자
**I want to** AI가 내 맞춤법이나 오타를 지적하지 않았으면 좋겠다
**So that** 꼽주는 느낌 없이 자유롭게 말할 수 있다

**수락 기준:**
- [ ] AI가 사용자의 오타/맞춤법/띄어쓰기를 언급하지 않는다
- [ ] 사용자의 표현을 "정확한" 표현으로 바꾸지 않는다
- [ ] 사용자 말을 인용할 때 원문 그대로 사용한다
- [ ] "~라고 하셨는데, ~인 거죠?" 식의 교정형 확인을 하지 않는다

## 세션 로그

### US-014: 과거 대화 목록 확인 (P1)

**As a** 여러 번 대화한 사용자
**I want to** 과거 대화 목록을 날짜별로 보고 싶다
**So that** 특정 대화를 다시 찾아볼 수 있다

**수락 기준:**
- [ ] 월별 그룹핑된 세션 리스트
- [ ] 각 카드: 날짜 + 입력 타입 요약 + 발견 텍스트 + 발견 개수
- [ ] 카드 탭 → `/chat/:sessionId`로 이동
- [ ] 무한 스크롤 (20개씩 로드)
- [ ] 대화 0회: "아직 대화한 적 없어요" + 홈 이동 버튼

---

### US-015: 과거 대화 다시 보기 (P1)

**As a** 과거 대화가 궁금한 사용자
**I want to** 대화 원문과 발견을 다시 보고 싶다
**So that** 그때 무슨 이야기를 했는지 떠올릴 수 있다

**수락 기준:**
- [ ] 읽기 전용 대화 뷰 (입력 영역 없음)
- [ ] 대화 원문 + 인사이트 카드 표시
- [ ] 하단: "이 대화 이어하기" 버튼 → `/chat`으로 전환
- [ ] 존재하지 않는 sessionId: "이 대화를 찾을 수 없어요" + 세션 목록 이동

## 프로필

### US-016: 에센스 프로필 확인 (P1)

**As a** 자신의 에센스가 궁금한 사용자
**I want to** 에센스 + 주제 모음 + 발견 목록을 한 화면에서 보고 싶다
**So that** 나에 대해 축적된 것들을 한눈에 파악할 수 있다

**수락 기준:**
- [ ] 에센스 카드 (한 줄 + 6축 collapsible + 태그 + 팔레트)
- [ ] 주제 모음 (세션 5개 이상일 때)
- [ ] 발견 목록 (최근 20개, 태그 필터)
- [ ] 반복 키워드 (3개 이상일 때)
- [ ] 에센스 한 줄 탭 → 상세 (원본 입력 항목 + 분석 일시)

---

### US-017: 에센스 공유 (P1)

**As a** 에센스가 마음에 든 사용자
**I want to** 카카오톡이나 인스타그램에 공유하고 싶다
**So that** 친구들에게 나의 에센스를 보여줄 수 있다

**수락 기준:**
- [ ] 에센스 카드 길게 누르기 → 공유 메뉴 바텀시트
- [ ] 카카오톡: 1:1 이미지 (720x720) - 에센스 한 줄 + 팔레트
- [ ] 인스타 스토리: 9:16 이미지 (1080x1920) - 사진 콜라주 + 에센스
- [ ] 이미지 저장: PNG 파일
- [ ] 링크 복사: 공유 페이지 URL (OG 이미지 포함)
- [ ] 공유는 에센스 결과 직후가 아니라, 대화 후 발견이 생긴 뒤 프로필에서만

---

### US-018: 주간 리포트 확인 (P2)

**As a** 꾸준히 대화하는 사용자
**I want to** 한 주간의 대화에서 발견된 패턴과 변화를 정리해서 보고 싶다
**So that** 내가 요즘 어떤 주제에 머물러 있는지 알 수 있다

**수락 기준:**
- [ ] 해당 주 세션 1개 이상 시 자동 생성 (일요일 자정)
- [ ] 내용: 대화 횟수 + 새 발견 + 반복 키워드 + 에센스 변화 + 한 줄 요약
- [ ] 프로필 하단 "이번 주" 섹션에 카드 형태로 표시
- [ ] 카드 탭 → 상세 바텀시트 (세션별 링크 포함)
- [ ] 수동 트리거도 가능

## 텔레그램 봇

### US-019: 텔레그램에서 대화 시작 (P1)

**As a** 텔레그램을 주로 쓰는 사용자
**I want to** 텔레그램에서 사진이나 글을 보내면 바로 대화가 시작되었으면 좋겠다
**So that** 웹앱을 따로 열지 않아도 된다

**수락 기준:**
- [ ] `/start` → 안내 메시지
- [ ] 에센스 미생성 + 항목 1개 → 자동 에센스 분석 → 결과 텍스트 전송 + 웹 링크
- [ ] 에센스 완료 후 사진/텍스트/링크 → 즉시 메타대화 시작
- [ ] 인사이트 감지 → 인사이트 텍스트 + `[웹에서 보기]` 인라인 버튼

### US-020: 텔레그램 엣지케이스 처리 (P1)

**As a** 텔레그램 봇 사용자
**I want to** 잘못된 입력을 해도 친절하게 안내받고 싶다
**So that** 뭘 보내야 하는지 헷갈리지 않는다

**수락 기준:**

- [ ] 음성 메시지 → "아직 음성은 읽을 수 없어요. 텍스트로 보내줄래요?"

## 공통

### US-021: 네트워크 오류 복구 (P0)

**As a** 불안정한 네트워크의 사용자
**I want to** 연결이 끊겨도 대화가 유실되지 않았으면 좋겠다
**So that** 다시 연결되면 이어서 할 수 있다

**수락 기준:**
- [ ] 오프라인 감지 시 상단 바: "연결이 끊겼어요"
- [ ] 전송 실패 메시지에 빨간 `!` 표시
- [ ] `!` 탭 → "다시 보낼까요?" → 수동 재전송
- [ ] 재연결 시 로컬 큐의 메시지 자동 전송
- [ ] "다시 연결됐어요" 2초 후 사라짐

### US-022: 인증 및 데이터 연동 (P0)

**As a** 텔레그램과 웹 둘 다 쓰는 사용자
**I want to** 어디서 대화하든 같은 데이터를 보고 싶다
**So that** 텔레그램에서 대화하고 웹에서 확인할 수 있다

**수락 기준:**
- [ ] 텔레그램 uid 기반 인증
