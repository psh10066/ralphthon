# 품질 평가 세트 (Eval Set)

> AI 출력물의 품질을 자동으로 검증하기 위한 테스트 케이스.
> 랄프루프에서 각 엔진의 출력을 이 기준으로 평가한다.

---

## 1. 에센스 분석 평가

### 1-1. 골든 셋 (입력 → 기대 방향)

| ID | 입력 사진 설명 | 기대 에센스 방향 | FAIL 조건 |
|----|---------------|----------------|-----------|
| E1 | 카페 나무 테이블, 도자기 접시, 빵, 아이스라떼, 핑크벽, 꽃그림 | 따뜻함, 유기적 질감, 불투명, 아늑한 공간 | "미니멀", "차가운", "기하학적" 키워드 |
| E2 | 돌로미티 초원, 풀장 너머 풍경, 뤽상부르 공원 기지개 | 열린 공간, 자연, 서늘~따뜻, 가벼운 무게 | "좁은", "실내", "무거운" 키워드 |
| E3 | 콘크리트 건물, 다크 카페, 가죽 가방, 검은 노트북, 위스키잔 | 묵직한, 어두운 톤, 날카로운/단단한 질감 | "부드러운", "파스텔", "가벼운" 키워드 |
| E4 | 흰 벽, 빈 방, 유리컵, 무인양품 선반, 하얀 리넨 | 비어있는, 미니멀, 투명/반투명, 가벼운 | "채워진", "화려한", "따뜻한 나무" 키워드 |

### 1-2. 일관성 테스트

같은 사진 세트를 3회 분석했을 때:
- **에센스 한 줄**: 완전히 같을 필요 없지만 **방향이 같아야** 함 (따뜻한/아늑한 계열)
- **6축**: 각 축의 방향(따뜻↔서늘, 무거운↔가벼운 등)이 3회 모두 동일
- **팔레트**: 톤(warm/cool)이 3회 모두 동일. hex값은 달라도 됨
- **FAIL**: 3회 중 1회라도 반대 방향이 나오면

### 1-3. 품질 체크리스트

| 기준 | PASS | FAIL |
|------|------|------|
| 에센스 한 줄 길이 | 15~20자 | 10자 미만 또는 30자 초과 |
| 에센스 한 줄 구체성 | "손 닿는 곳의 아늑함을 모으는 사람" | "따뜻한 사람", "자연을 좋아하는 사람" |
| 6축 각각 한 줄 | 감각적 언어로 서술 | 형용사 나열 ("따뜻하고 부드러운") |
| 첫 질문 = 열린 호기심 | "사진이 전부 가까이에 있는 것들이거든요. 뭐가 끌렸어요?" | "혼자 쉬고 싶은 상태 아닌가요?" (심리 진단) |
| 첫 질문에 가설 없음 | 관찰 사실 + "뭐가 끌렸어요?" | "~아닌가요?" (첫 턴에서 가설) |
| 관찰 vs 진단 | "이런 것에 끌린다" | "당신은 이런 유형이다" |
| 사진 내용물 vs 공통점 | "선택한 것들의 공통점을 읽음" | "피자를 좋아하시네요" |
| 겸손한 톤 | "사진에서 이렇게 읽어봤어요" | "당신의 에센스입니다" |

## 2. 메타대화 평가

### 2-1. 질문 품질 골든 셋

| ID | 입력 | 입력 타입 | 좋은 질문 예시 | FAIL 질문 예시 |
|----|------|----------|--------------|---------------|
| D1 | "속도를 늦춰야 빨라진다" 글 | article | "이 글에서 '속도를 늦춰야'가 걸렸다면, 지금 뭘 빠르게 하려다 막혀있어?" | "이 글은 속도와 효율성에 대한 내용이네요" |
| D2 | 풍경 사진 1장 | photo | "왜 풍경만 올렸어? 색감이 좋아? 구도가 좋아? 그때 기억이 좋아?" | "아름다운 풍경이네요! 어디에서 찍었나요?" |
| D3 | "요즘 팀한테 기획 공유가 자꾸 늦어진다" 메모 | memo | "이 생각이 왜 '지금' 떠올랐어? 오늘 뭐가 있었어?" | "기획 공유가 늦어지는 이유를 분석해볼게요" |
| D4 | 카페 사진 + 에센스 "아늑함을 모으는 사람" | photo | "또 카페네. 에센스에서 '손 닿는 곳의 아늑함'이라고 읽혔는데, 진짜 그래? 카페가 그 '손 닿는 곳'인 거야?" | "커피를 좋아하시나 봐요!" |
| D5 | 퇴사 고민 글 | article | "'떠나고 싶다'가 진짜야, 아니면 '여기서는 안 된다'가 진짜야?" | "퇴사는 신중하게 결정해야 합니다" |

### 2-2. 질문 분류 테스트

모든 AI 질문을 다음 중 하나로 분류:
- **탐색**: "왜?", "뭐가 걸렸어?", "어떤 느낌?" → PASS
- **역추적**: "그게 왜 걸렸을까?", "지금 상황과 연결되나?" → PASS
- **연결**: "지난번에도 비슷한 말 했는데", "에센스의 ~와 닿는데" → PASS
- **요약**: "이 글의 핵심은~", "정리하면~" → FAIL
- **진단**: "당신은 ~한 유형", "~해야 합니다" → FAIL
- **일상 대화**: "맛있겠다!", "좋은 곳이네요" → FAIL

**PASS 기준**: 대화 10턴 중 요약/진단/일상 대화 0개

### 2-3. 에센스 연결 품질

| 기준 | PASS | FAIL |
|------|------|------|
| 연결 타이밍 | 대화 3턴 이후, 자연스러운 맥락에서 | 첫 턴에서 바로 연결 |
| 연결 표현 | "에센스에서 ~라고 했는데, 여기도 닿는 것 같아" | "당신의 에센스는 ~이므로" |
| 연결 빈도 | 세션당 1~2회 | 매 턴마다 연결 |
| 억지 연결 | 사용자가 "맞아"로 반응할 수 있는 수준 | 관련 없는 축에 억지 매핑 |

## 3. 축적 엔진 평가

### 3-1. 패턴 감지 골든 셋

| ID | 인사이트 로그 (3개 이상) | 기대 패턴 | FAIL |
|----|------------------------|----------|------|
| A1 | "닫는 게 안 돼", "준비 안 된 채로 꺼내기", "마무리 근육" | "만드는 건 빠른데 닫는 게 어렵다" | 각각 별개 패턴으로 처리 |
| A2 | "혼자 채우고 나서야 꺼냄", "팀한테 공유가 늦어짐", "무능해 보일까봐" | "혼자 완성하려는 패턴" | 키워드만 나열 |
| A3 | "카페 사진", "빵 사진", "집에서 빵" | "음식/카페에서 에너지 충전" | "빵을 좋아함" |

### 3-2. 지형도 클러스터링 품질

| 기준 | PASS | FAIL |
|------|------|------|
| 클러스터 라벨 | 사용자의 언어에서 가져옴 | AI가 만든 추상적 라벨 |
| 클러스터 수 | 인사이트 10개당 2~4개 | 인사이트 수 = 클러스터 수 (각각 1개) |
| 클러스터 최소 크기 | 3개 이상 인사이트 | 1개짜리 클러스터 존재 |
| 연결 | 의미적으로 관련 있는 클러스터 연결 | 모든 클러스터가 연결되거나 전부 분리 |

### 3-3. 시간 흐름 평가

| 기준 | PASS | FAIL |
|------|------|------|
| 에센스 변화 감지 | 새 사진 분석 시 이전과 비교, 변화 서술 | 매번 독립적으로 분석 |
| 패턴 생애주기 | 발견 → 확인(3회+) → 상태 변경 추적 | confirmedCount만 증가 |
| 리포트 시간 비교 | "지난주 대비 ~가 새로 올라왔다" | 현재 상태만 나열 |

## 4. UI/UX 평가 (수동)

랄프루프 완료 후 수동 체크:

| 기준 | PASS | FAIL |
|------|------|------|
| 랜딩 → 에센스 결과 | 3분 이내 도달 | 5분 초과 |
| 사진 업로드 | 드래그앤드롭 + 클릭 모두 동작 | 한쪽만 동작 |
| 모바일 375px | 모든 화면 깨지지 않음 | 가로 스크롤 발생 |
| 에센스 결과 → "이게 나야?" | 에센스 한 줄이 화면 중심, 여백 충분 | 정보 과밀 |
| 대화 UI | 버블 구분, 인사이트 카드 하이라이트 | 평문 나열 |
| 지형도 | 클러스터 시각화, 클릭 시 상세 | 텍스트 리스트만 |
| 공유 이미지 | 카카오/다운로드 동작 | 에러 |

## 5. 통합 테스트 시나리오

### 시나리오 1: 첫 사용자 온보딩

```
1. 랜딩 페이지 접속
2. 예시 카드 3개 확인 → 모달 열림
3. "시작하기" 클릭
4. 사진 5장 업로드
5. 로딩 4단계 표시
6. 에센스 결과 표시 (한 줄 + 6축 + 팔레트 + 관찰 + 열린 질문)
7. 열린 질문에 자유 응답
8. 즉시 `/chat`으로 대화 진입 (에센스 결과가 끝이 아니라 첫 대화 턴)
9. 글 링크 입력
10. AI 질문 → 사용자 답변 → 발견 → 저장
11. 프로필에서 에센스 + 발견 1개 확인
```

**PASS 기준**: 1~11 전체가 에러 없이 동작. 에센스/질문 품질은 위 eval 기준 충족.

### 시나리오 2: 텔레그램 → 웹

```
1. 텔레그램에 사진 1장 전송
2. 봇이 질문 ("이 사진에서 뭐가 끌려?")
3. 사용자 답변
4. 봇이 역추적 질문
5. 인사이트 감지 → "웹에서 확인해보세요" + 링크
6. 웹 프로필에서 해당 인사이트 확인
```

### 시나리오 3: 축적 (5세션 후)

```
1. 5개 세션 완료 (사진 2 + 글 2 + 메모 1)
2. 프로필에서 지형도 확인 — 클러스터 2개 이상
3. 패턴 1개 이상 감지됨
4. 세션 로그에서 과거 대화 확인 가능
5. 에센스에 변화가 있으면 변화 표시
```

## 6. 자동 평가 구현 (랄프루프용)

### eval 스크립트 구조

```typescript
// /tests/eval-essence.ts
// 골든 셋 E1~E4에 대해 에센스 분석 API를 호출하고
// FAIL 조건 키워드가 포함되면 실패

// /tests/eval-dialogue.ts
// 골든 셋 D1~D5에 대해 대화 API를 호출하고
// 질문 분류(탐색/역추적/연결 vs 요약/진단/일상)를 판정
// 판정은 별도 LLM 호출로 자동 분류

// /tests/eval-accumulation.ts
// 골든 셋 A1~A3의 인사이트 로그를 넣고
// 패턴 감지 결과가 기대와 맞는지 확인

// /tests/eval-integration.ts
// 시나리오 1~3을 자동으로 수행
```

### 평가 LLM 프롬프트 (질문 분류용)

```
다음 AI 질문을 분류하세요:

질문: "{question}"

카테고리:
- 탐색: 열린 질문, "왜?", "뭐가 걸렸어?"
- 역추적: "왜 그게 걸렸을까?", 현재 상황으로 거슬러 올라감
- 연결: 이전 대화/에센스와 연결
- 요약: 글이나 상황을 정리/요약
- 진단: "당신은 ~한 사람", 판정
- 일상대화: "맛있겠다", "좋은 곳이네요"

출력: { "category": "탐색", "reason": "..." }
```
