# 아이디어 발전 타임라인

이 서비스가 어떻게 여기까지 왔는지. 시간순.

### 배경: 메타대화 28세션 (2026-02 ~ 2026-03)
- 보경이 Claude와 메타대화 스킬을 28세션 사용
- 글을 가져오면 요약이 아니라 질문을 던지고, 자기 이해로 연결
- "글은 촉매였고, 발견은 나 안에서 일어났다"
- 이걸 서비스로 만들 수 있을까? → 랄프톤 해커톤 참가 결심

### v1: "글을 더 읽는 게 아니라, 나를 더 아는 것" (2026-03-28)
- 글 읽기 FOMO를 해소하는 서비스로 시작
- 핵심: 글을 넣으면 요약이 아니라 질문이 온다
- 축적이 해자 — 30회 쌓이면 대체 불가능

### 전환 1: "취향도 자기 이해의 한 축" (2026-03-28)
- 누군가 피치 문서를 보고 프로토타입을 만들어옴 → 후킹이 부족
- vir-tually.love 참조 → "첫 10초에 잡아야 한다"
- 취향 UI 생성 아이디어 → "취향으로 자꾸 가네" 자각 → "취향도 자기 이해니까 괜찮아"
- 사진 5장 → AI가 에센스를 읽음 → "이게 나야?" 순간

### 전환 2: "에센스 언어가 wow, UI 테마가 아님" (2026-03-28~29)
- 실제 사진으로 파이프라인 테스트 (보경 사진 5장 + 여행 3장)
- CSS 테마 생성 → 시각적 차이 체감 약함
- "손 닿는 곳의 아늑함을 모으는 사람" vs "열린 곳에서 혼자 숨 쉬기" — 언어가 사람을 잡음
- 배민 디자이너의 "에센스" 개념 차용 → 당시 6축 (양감/질감/투명도/촉각/무게/온도) 도입 (이후 자유 형식 읽기 + 태그 방식으로 단순화)

### 전환 3: "FOMO와 호기심은 다른 감정" (2026-03-29)
- 퍼널에 FOMO(글 저장)와 호기심(사진)이 섞여있음 발견
- "글, 또 저장했죠?" → "사진 올려봐"는 감정이 안 이어짐
- 분리: 입구 = 호기심(사진), 깊이 = FOMO 해소(글)

### 전환 4: "모든 입력에 즉시 호기심 질문" (2026-03-29)
- 메타대화가 "에센스 후 별도 단계"가 아니라 모든 입력에 즉시 반응
- 풍경 사진 하나 올려도 → "왜 풍경만 올렸어? 색감? 구도? 그때 기억?"
- 사진이든 글이든 메모든 → 바로 호기심 질문 시작

### 전환 5: "대화는 어디서든, 축적은 한 곳에서" (2026-03-29)
- 텔레그램에서 시작할 수 있어야 함
- 모든 채널에서 대화 시작 → 웹앱은 축적이 잘 보이는 곳
- 주제 모음: 비슷한 읽기끼리 묶여서 켜가 두꺼워진다

### 전환 6: "거울/지형도 제거, 주제 클러스터로" (2026-03-29)
- 와이어프레임 v3 리뷰에서 거울/지형도 컨셉이 과하다는 피드백
- 사진 5장 고정 → 뭐든 하나 drop으로 변경
- 6축 비주얼 분석 wow point 제거 → 내용+무드 같이 읽은 인사이트가 핵심 (6축 고정 구조 폐기)
- 대화 강제 → 반응하면 대화, 안 하면 그냥 쌓임
- 세션 로그(날짜별) → 주제 모음(주제별 클러스터)으로 축적 구조 변경
- 지형도 등고선 맵 → 주제 칩/리스트로 단순화
