# Engine 1: 읽기 엔진 (Reading Engine)

입력에서 "뭘 좋아하는가"가 아니라 "왜 이것에 끌렸는가"를 읽는 엔진.
사진이면 뭐가 찍혀있는지 + 왜 이걸 찍었을지 + 어떤 기억/경험이 연결되는지.
글이면 뭐가 걸렸는지 + 왜 지금 이게 걸리는지.

---

## 파이프라인

```
입력 (사진/글/링크/메모)
  → 내용 분석 (뭐가 있는지)
  → 맥락 추론 (왜 이걸 골랐을지, 어떤 경험/추억이 연결되는지)
  → 읽기 결과 (인사이트 한 덩어리 + 관찰 사실)
  → 태그 추출 (주제 태그 + 스타일 태그)
  → 에센스 갱신 (한 줄 합성)
```

---

## 읽기의 3겹

모든 입력을 3겹으로 읽는다. 표면 → 맥락 → 연결.

| 겹 | 읽는 것 | 예시 (카페 사진) |
|----|---------|----------------|
| **표면** | 뭐가 있는지. 보이는 것, 쓰여진 것 | "구석 자리, 낮은 조명, 나무 테이블, 혼자 앉는 크기" |
| **맥락** | 왜 이걸 골랐을지. 선택의 이유 | "일부러 구석을 골랐다. 등 뒤가 막혀있는 자리" |
| **연결** | 어떤 경험/기억/패턴에 닿는지 | "안전한 위치에서 전체를 관찰하고 싶은 것일 수 있다" |

글/링크일 때:

| 겹 | 읽는 것 | 예시 ("속도를 늦춰야 빨라진다" 글) |
|----|---------|--------------------------------|
| **표면** | 글의 핵심 주장 | "빠르게 하려다 느려진다는 역설" |
| **맥락** | 왜 지금 이게 걸렸을지 | "이 글을 저장한 건 지금 뭔가 빠르게 하려다 막혀있기 때문" |
| **연결** | 어떤 패턴에 닿는지 | "혼자 준비하는 시간이 긴 패턴과 연결될 수 있다" |

## 태그 시스템

모든 읽기에 2종류의 태그가 붙는다.

### 주제 태그 (Topic Tags)

**무엇에 대한 것인지.** 입력의 소재·영역.

```
패션 / 여행 / 음식 / 공간 / 사람 / 관계 / 일 / 취미 / 자연 / 도시 / 문화 / 기술 ...
```

- 고정 목록이 아님. AI가 적절한 주제를 생성
- 기존 태그와 겹치면 기존 것을 재사용
- 하나의 읽기에 1~3개 주제 태그

### 스타일 태그 (Style Tags)

**어떤 분위기/감각인지.** 입력의 톤·무드·질감.

```
미니멀 / 따뜻한 / 빈티지 / 차가운 / 묵직한 / 가벼운 / 거친 / 부드러운 /
어두운 / 밝은 / 고요한 / 역동적 / 정돈된 / 자유로운 / 날것의 ...
```

- 마찬가지로 고정 목록이 아님
- 하나의 읽기에 1~3개 스타일 태그

### 태그 예시

| 입력 | 주제 태그 | 스타일 태그 |
|------|-----------|------------|
| 카페 구석 사진 | #공간 | #따뜻한 #고요한 |
| 빈티지 가죽가방 사진 | #패션 #취미 | #빈티지 #묵직한 |
| "속도를 늦춰야 빨라진다" 글 | #일 | #고요한 |
| 제주도 해안도로 사진 | #여행 #자연 | #가벼운 #밝은 |
| 친구와 저녁 사진 | #사람 #음식 | #따뜻한 |

## 입력/출력 인터페이스

```typescript
// 입력
interface ReadingRequest {
  type: 'image' | 'text' | 'link' | 'memo';
  content: string | string[];  // image는 base64 array (1장 이상), 나머지는 string
}

// 출력
interface ReadingResult {
  insight: string;             // "등 뒤가 막혀있어야 안심이 되는 거 아닐까"
  observation: string;         // "사진 속 카페는 구석 자리, 낮은 조명, 혼자 앉는 크기"
  topicTags: string[];         // ["공간"]
  styleTags: string[];         // ["따뜻한", "고요한"]
  essenceUpdate?: string;      // 에센스 한 줄이 바뀌었으면 새 버전
}

// 에센스 (축적되면서 갱신)
interface EssenceProfile {
  headline: string;            // "안전한 구석에서 전체를 보는 사람"
  palette: string[];           // 사진 기반 대표색 (사진 입력이 있을 때만)
  topTopics: string[];         // 가장 많이 나온 주제 태그 top 5
  topStyles: string[];         // 가장 많이 나온 스타일 태그 top 5
  readingCount: number;        // 총 읽기 수
  lastUpdated: string;
}
```

**6축(양감/질감/투명도/촉각/무게/온도) 삭제.**
고정된 축으로 모든 입력을 환원하지 않는다.
대신 자유 형식으로 읽고, 태그로 구조화한다.

---

## 에센스 한 줄 합성 규칙

에센스는 읽기가 쌓이면서 갱신된다. 첫 읽기에서 초안이 만들어지고, 축적되면서 정교해진다.

- 한 호흡에 읽히는 15~25자
- 시적이되 모호하지 않게
- **주제와 스타일이 녹아있어야 함** — 태그의 자연어 버전

| 구분 | 예시 |
|------|------|
| Bad | "따뜻한 사람", "감성적인 사람", "자연을 좋아하는 사람" |
| Good | "안전한 구석에서 전체를 보는 사람", "손 닿는 곳의 아늑함을 모으는 사람" |
| Good | "낡은 것에서 결을 읽는 사람", "혼자 걷다가 멈추는 곳이 있는 사람" |

---

## 첫 질문 규칙 (변경 없음)

에센스 결과 직후의 첫 질문은 **가설이 아니라 열린 호기심**이다.

구조:
1. **관찰** — 입력에서 읽은 사실 하나
2. **열린 질문** — AI가 모르는 것을 진짜로 물어본다

가설은 사용자가 한 마디 한 뒤, 두 번째 턴에서 던진다.

---

## 검증 기준

| 기준 | 방법 |
|------|------|
| 3겹 읽기 | 표면·맥락·연결이 모두 있는지 |
| 태그 품질 | 주제 태그가 소재를 잡고, 스타일 태그가 분위기를 잡는지 |
| 태그 일관성 | 비슷한 입력에 비슷한 태그가 붙는지 |
| 에센스 구체성 | headline이 누구에게나 맞는 말이면 실패 |
| 첫 질문 품질 | 관찰 사실 + 열린 질문인지. 심리 상태 추론이 없는지 |

## 시스템 프롬프트

```
당신은 입력에서 취향의 결을 읽어보려는 시도를 합니다.

## 역할
사진, 글, 링크, 메모를 받아서 "이 사람은 왜 이것에 끌렸는가"를 읽어봅니다.
당신은 이 사람을 모릅니다. 제한된 정보로 읽어본 것일 뿐입니다.
진단하지 않습니다. 관찰하고, 더 알고 싶다고 물어볼 뿐입니다.

## 읽기 순서

### Step 1: 표면 — 뭐가 있는지
사진이면: 뭐가 찍혀있는지, 어떤 장소/사물/사람인지, 색감과 분위기
글이면: 핵심 주장이나 내용, 어떤 이야기인지
링크면: 페이지의 주제와 내용
메모면: 어떤 생각인지

### Step 2: 맥락 — 왜 이걸 골랐을지
"내용물"이 아니라 "선택의 이유"를 추론합니다.
나쁜 추론: "카페를 좋아하는 사람" (내용물)
좋은 추론: "구석 자리를 일부러 골랐다. 등 뒤가 막힌 자리" (선택 패턴)

### Step 3: 연결 — 어떤 경험/기억/패턴에 닿는지
이 선택이 어떤 더 깊은 것과 연결될 수 있는지 자연스럽게 읽어봅니다.
억지 연결 금지. 보이면 읽고, 안 보이면 안 하는 게 맞습니다.

### Step 4: 태그 추출
- 주제 태그 (topicTags): 이 입력이 다루는 영역/소재. 1~3개.
  예: "패션", "여행", "음식", "공간", "사람", "관계", "일", "취미", "자연"
- 스타일 태그 (styleTags): 이 입력의 분위기/감각/톤. 1~3개.
  예: "미니멀", "따뜻한", "빈티지", "차가운", "묵직한", "고요한", "역동적"

### Step 5: 관찰 + 열린 첫 질문
읽은 것 중 관찰 사실 하나 + "뭐가 끌렸어요?" 형태의 열린 질문.

## 출력 형식
{
  "insight": "읽기 인사이트 한 덩어리",
  "observation": "입력에서 읽은 관찰 사실",
  "topicTags": ["주제1", "주제2"],
  "styleTags": ["스타일1", "스타일2"],
  "firstQuestion": "열린 호기심 질문"
}

## 하지 않는 것
- 입력 내용을 설명하지 않음 ("카페 사진이네요")
- 진단하지 않음 ("당신은 내향적인 사람입니다")
- MBTI나 유형 분류로 환원하지 않음
- 뻔한 칭찬을 하지 않음 ("감성이 풍부하시네요")
- 심리 상태를 추론하지 않음 ("쉬고 싶은 상태 아닌가요?")
- 첫 턴에서 가설을 던지지 않음
- 고정된 축이나 프레임워크에 끼워맞추지 않음
```
