# Engine 1: 에센스 분석 (Essence Analyzer)

사진에서 "뭘 찍었는가"가 아니라 "왜 이것에 끌렸는가"를 읽는 엔진.

---

## 파이프라인

```
사진 N장 → Vision Model 개별 분석 → 사진 간 교집합 → 6축 에센스 → 팔레트 → 한 줄 합성 → 가설 질문
```

### Step 1: 개별 사진 분석
각 사진에서 지배적 색감(hex), 공간감, 질감, 빛, 무드 키워드 3개를 읽는다.

### Step 2: 교집합 추출
사진 "내용물"이 아니라 "선택의 공통점"을 찾는다.

- Bad: "카페 사진이 많다" (내용물)
- Good: "낮은 조도, 나무 질감, 혼자 앉을 수 있는 구석" (선택 패턴)

검증 질문: "이 사람에게 사진 한 장을 더 보여준다면, 어떤 사진에 끌릴까?"

### Step 3~6: 아래 6축 → 팔레트 → 한 줄 → 가설 순서로 처리.

---

## 6축 정의

| 축 | 읽는 것 | 표현 예시 |
|----|---------|----------|
| 양감(Volume) | 채워짐/비워짐, 밀도 | "채워진 둥근 볼륨" |
| 질감(Texture) | 표면의 물성 | "도자기 유약, 나무결" |
| 투명도(Opacity) | 빛의 투과/차단 | "불투명. 빛을 품는 물질들" |
| 촉각(Tactility) | 만지면 어떤 느낌일지 | "따뜻하다" |
| 무게(Weight) | 존재감의 무게 | "탁자 위에 올려진 존재감" |
| 온도(Temperature) | 차가움-따뜻함 스펙트럼 | "미지근 ~ 따뜻" |

각 축은 형용사 한 단어가 아니라 **장면이나 촉감**으로 표현한다.

## 입력/출력 인터페이스

```typescript
// 입력
interface AnalyzeRequest {
  images: string[];  // base64, 5장
}

// 출력
interface EssenceProfile {
  headline: string;          // "손 닿는 곳의 아늑함을 모으는 사람"
  dimensions: {
    volume: Dimension;
    texture: Dimension;
    opacity: Dimension;
    tactility: Dimension;
    weight: Dimension;
    temperature: Dimension;
  };
  palette: string[];         // ["#C4A882", "#D4A5A5", ...]
  observation: string;       // "사진이 전부 가까이에 있는 것들이거든요" 관찰 사실
  firstQuestion: string;     // "뭐가 끌렸어요?" 열린 호기심 질문
  rawAnalysis: ImageAnalysis[];
}

interface Dimension {
  label: string;       // "양감"
  description: string; // "채워진 둥근 볼륨"
}
```

## 에센스 한 줄 합성 규칙

- 한 호흡에 읽히는 15~20자
- 시적이되 모호하지 않게
- 읽고 "아, 그 느낌" 하고 떠오르는 게 있어야 함

| 구분 | 예시 |
|------|------|
| Bad | "따뜻한 사람", "감성적인 사람", "자연을 좋아하는 사람" |
| Good | "손 닿는 곳의 아늑함을 모으는 사람", "빛이 머무는 자리를 찾는 사람" |

## 첫 질문 규칙 (가설이 아니라 호기심)

에센스 결과 직후의 첫 질문은 **가설이 아니라 열린 호기심**이다. AI가 모르는 걸 진짜로 물어본다.

구조:
1. **관찰** — 사진에서 읽은 사실 하나 (감각 패턴)
2. **열린 질문** — "뭐가 끌렸어요?" (AI가 모르는 것)

가설은 사용자가 한 마디 한 뒤, 두 번째 턴에서 던진다.

| 턴 | 역할 | 예시 |
|----|------|------|
| 첫 턴 (에센스 결과 직후) | 관찰 + 열린 질문 | "사진이 전부 가까이에 있는 것들이거든요. 뭐가 끌렸어요 — 질감? 색감? 그때 기억?" |
| 두 번째 턴 (사용자 응답 후) | 의도 + 가설 | "따뜻한 톤에 끌린다면, 혹시 차가운 곳에서는 불편해요? 아니면 따뜻한 걸 일부러 모으는 거예요?" |

- Bad (첫 턴에서 상태 진단): "손 닿는 데서 쉬고 싶은 거 아닌가요?"
- Bad (첫 턴에서 성격 점프): "혼자만의 시간에서 에너지를 충전하는 것 — 아닌가요?"
- Good (첫 턴 — 열린 호기심): "사진이 전부 손 닿는 거리에 있는 것들이거든요. 뭐가 끌렸어요?"
- Good (두 번째 턴 — 가설): "만지고 싶은 느낌에 끌린다면, 질감이 중요한 사람일 수 있어요. 옷이나 가구 고를 때도 만져보고 고르는 편이에요?"

## 검증 기준

| 기준 | 방법 |
|------|------|
| 일관성 | 같은 사진 3회 분석 → 6축 방향이 비슷한지 |
| 역추론 | 프로필만 보고 "어떤 사진을 골랐을까?" → 원본과 비교 |
| 구체성 | headline이 누구에게나 맞는 말이면 실패 |
| 첫 질문 품질 | 관찰 사실 + 열린 질문인지. 심리 상태 추론이 없는지 |

## 시스템 프롬프트

```
당신은 사진에서 취향의 에센스를 읽어보려는 시도를 합니다.

## 역할
사진 N장을 받아서 "이 사람은 왜 이것들에 끌렸는가"를 6축으로 읽어봅니다.
당신은 이 사람을 모릅니다. 사진이라는 제한된 정보로 읽어본 것일 뿐입니다.
진단하지 않습니다. 관찰하고, 더 알고 싶다고 물어볼 뿐입니다.

## 분석 순서

### Step 1: 개별 사진 분석
각 사진에서 지배적 색감(hex), 공간감, 질감, 빛, 무드 키워드 3개를 읽으세요.

### Step 2: 교집합 추출
"내용물"이 아니라 "선택의 공통점"을 찾으세요.
나쁜 교집합: "카페 사진이 많다"
좋은 교집합: "낮은 조도, 나무 질감, 혼자 앉을 수 있는 구석"

### Step 3: 6축 에센스
양감/질감/투명도/촉각/무게/온도. 형용사가 아니라 장면으로 표현.

### Step 4: 컬러 팔레트
전체 사진의 톤을 대표하는 지배색 5개 (hex).

### Step 5: 에센스 한 줄
15~20자. 시적이되 모호하지 않게.
나쁜 예: "따뜻한 사람" / 좋은 예: "손 닿는 곳의 아늑함을 모으는 사람"

### Step 6: 관찰 + 열린 첫 질문
사진에서 읽은 감각 패턴 사실 하나 + "뭐가 끌렸어요?" 형태의 열린 질문.
감각 → 심리 상태로 점프하지 않는다. 끌림의 방식을 물어볼 뿐.

## 출력 형식
{
  "headline": "에센스 한 줄",
  "dimensions": { "volume": {...}, "texture": {...}, ... },
  "palette": ["#hex1", ...],
  "observation": "사진에서 읽은 관찰 사실 한 문장",
  "firstQuestion": "열린 호기심 질문"
}

## 하지 않는 것
- 사진 내용을 설명하지 않음 ("카페 사진이네요")
- 진단하지 않음 ("당신은 내향적인 사람입니다")
- MBTI나 유형 분류로 환원하지 않음
- 뻔한 칭찬을 하지 않음 ("감성이 풍부하시네요")
- 심리 상태를 추론하지 않음 ("쉬고 싶은 상태 아닌가요?")
- 첫 턴에서 가설을 던지지 않음 (가설은 사용자 응답 이후)
```
