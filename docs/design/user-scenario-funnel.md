# droppi — 퍼널 유저 시나리오

> 기획/개발이 이 문서만 보고 구현할 수 있는 수준의 퍼널별 시나리오.
> 각 단계마다 화면, 유저 행동, 시스템 반응, API 호출, 성공 기준을 명시한다.

---

## 퍼널 구조 요약

```
F1. 유입 (Landing)          → "이게 뭐지?"
F2. 온보딩 (Upload)         → "사진 5장 올려볼까"
F3. WOW (First Drop)        → "아, 이게 나야?"    ← 전환점
F4. 공유 (Share)            → "이거 봐봐"          ← 바이럴 루프
F5. 재방문 (Home)           → "오늘은 이거 가져왔어"
F6. 대화 (Meta-Dialogue)    → "아, 그래서 그랬구나"
F7. 축적 (Accumulation)     → "나 변했네"          ← 해자
```

## F1. 유입 — "이게 뭐지?"

### 진입 경로

| 경로 | 유저가 보는 것 | 감정 |
|------|--------------|------|
| A. 카카오톡 공유 카드 | 친구의 에센스 한 줄 + 팔레트 | "뭐야 이거, 나도 해볼래" |
| B. 인스타 스토리 | 에센스 카드 + 사진 콜라주 | "예쁘다, 나도" |
| C. 직접 URL | 랜딩 페이지 | "뭐하는 서비스지?" |

### 화면: 랜딩 (`/`)

```
[전체 화면, 미색 배경 — 코튼 캔버스]

                    droppi


        사진 5장을 드롭해 보세요.


              [ drop ]
```

**설명 없음.** 호기심만. "뭐 하는 건데?" → 눌러봄.
빈 캔버스. 설명 안 한다.

### 스펙

| 항목 | 값 |
|------|-----|
| 헤드라인 | "droppi" — Noto Serif KR 36px/500, `#3D3632` |
| 서브카피 | "사진 5장을 드롭해 보세요." — Pretendard 16px/300, `#8A8480` |
| CTA | "drop" — Pretendard 16px/400, `#3D3632`, 밑줄, 배경 없음 |
| 배경 | `#FFFFFF` (순백 아닌 약간의 온기) |
| 여백 | 화면 중앙 정렬, 수직 중앙보다 살짝 위 (40%) |

### 성공 기준

| 지표 | 목표 |
|------|------|
| 랜딩 → drop 클릭 | 70%+ (설명이 없으니 이탈 적음) |
| 평균 체류 | 5초 이내 (빠르게 넘어가야 함) |

## F2. 온보딩 — "사진 5장 올려볼까"

### 유저 행동 시퀀스

```
1. "drop" 탭
2. 사진 업로드 화면 등장
3. 갤러리에서 사진 선택 (1장씩, 최대 5장)
4. 5장 채우면 "읽기 시작" 버튼 활성화
5. 탭 → 로딩으로 전환
```

### 화면: 업로드 (랜딩 내 스크롤 or 모달)

```
[상단]
끌리는 사진을 골라주세요.
예쁠 필요 없어요.

[그리드 — 2x3, 마지막 칸은 안내]
┌────┐ ┌────┐
│ +  │ │ +  │
└────┘ └────┘
┌────┐ ┌────┐
│ +  │ │ +  │
└────┘ └────┘
┌────┐
│ +  │   0/5
└────┘

[하단 — 5장 채우면 활성화]
[ 읽기 시작 ]  (비활성: #C4C0BC / 활성: #3D3632)
```

### 마이크로카피

| 시점 | 카피 |
|------|------|
| 0장 | "끌리는 사진을 골라주세요. 예쁠 필요 없어요." |
| 1~4장 | "{n}/5" (카운터만) |
| 5장 | "다 골랐으면, 읽어볼게요." |
| 유사 사진 3장+ | 토스트: "비슷한 사진이 많아요. 다른 상황의 사진도 넣으면 더 풍부하게 읽을 수 있어요." |

### 스펙

| 항목 | 값 |
|------|-----|
| 그리드 | 2열, 간격 12px |
| 썸네일 | 정사각, 라운드 8px, object-fit: cover |
| + 아이콘 | 40x40, stroke 1.5px, `#C4C0BC` |
| 삭제 | 썸네일 우상단 X 버튼 (탭하면 해당 사진 제거) |
| 이미지 압축 | 클라이언트에서 1024px max, quality 0.8, base64 |

### 에러 처리

| 케이스 | 대응 |
|--------|------|
| 텍스트 스크린샷 업로드 | "이건 글이네요! 텍스트로 넣으면 더 잘 읽을 수 있어요." (에센스 분석에선 무시하지 않고 진행) |
| 이미지 업로드 실패 | 자동 재시도 1회 → 실패 시 "다시 시도해주세요" + 재시도 버튼 |
| 5장 미만으로 "읽기 시작" 시도 | 버튼 비활성. 탭해도 반응 없음 |

### 성공 기준

| 지표 | 목표 |
|------|------|
| 업로드 시작 → 5장 완료 | 80%+ |
| 5장 선택 소요시간 | 90초 이내 |
| 유사사진 토스트 후 교체율 | 측정만 (강제 아님) |

## F3. WOW — "아, 이게 나야?"

> 전체 서비스의 전환점. 여기서 반응이 안 오면 공유도 재방문도 없다.

### 유저 경험 시퀀스

```
1. "읽기 시작" 탭
2. 로딩 애니메이션 (4단계, ~12초)
3. 에센스 화면 등장
4. 에센스 한 줄을 읽는다 → (여기서 반응이 갈린다)
5. 6축 분석을 천천히 스크롤
6. 팔레트를 본다
7. 가설 질문을 읽는다
8. 반응 선택 (맞아요 / 좀 다른데 / 잘 모르겠어요)
9. 공유 버튼 노출
```

### 화면: 로딩

```
[세로 중앙]

  ✓  색감을 보고 있어요
  ✓  질감을 느끼고 있어요
  ◉  온도를 재고 있어요        ← 현재 (펄스)
  ○  언어로 옮기고 있어요
```

| 단계 | 텍스트 | 상태 | 시간 |
|------|--------|------|------|
| 1 | "색감을 보고 있어요" | 완료 (✓) | 3초 |
| 2 | "질감을 느끼고 있어요" | 완료 (✓) | 3초 |
| 3 | "온도를 재고 있어요" | 진행 중 (◉ 펄스) | 3초 |
| 4 | "언어로 옮기고 있어요" | 대기 (○) | 3초 |

**API 호출**: `POST /api/analyze` — 로딩 시작과 동시에 호출. 12초 타이머와 API 응답 중 늦는 쪽 기준으로 전환.

### 화면: 에센스 (WOW 화면)

```
[풀스크린, 미색 배경, 스크롤]

─────────────────────────────────

  "손 닿는 곳의 아늑함을
   모으는 사람"

─────────────────────────────────

  양감    채워진 둥근 볼륨
  ████████░░░░░░░░

  질감    도자기 유약, 나무결
  ██████░░░░░░░░░░

  투명도  불투명. 빛을 품는 물질들
  ██████████░░░░░░

  촉각    따뜻하다
  ████████████░░░░

  무게    탁자 위에 올려진 존재감
  ██████░░░░░░░░░░

  온도    미지근 ~ 따뜻
  ██████████░░░░░░

─────────────────────────────────

  ● ● ● ● ●
  (팔레트 5색)

─────────────────────────────────

  "혼자만의 시간에서 에너지를
   충전하는데, 그 '혼자'가
   좋아하는 것에 둘러싸여
   무언가 하고 있는 것
   — 아닌가요?"

─────────────────────────────────

  [ 맞아요 ]  [ 좀 다른데 ]  [ 잘 모르겠어요 ]

  카카오톡으로 공유    인스타 스토리 공유

─────────────────────────────────
```

### 반응별 분기

| 반응 | 시스템 동작 | 다음 화면 |
|------|-----------|----------|
| **맞아요** | 에센스 확정 저장 → 공유 CTA 강조 | 공유 → 홈 |
| **좀 다른데** | "어디가 달라요?" 텍스트 입력 → 첫 메타대화 시작 | 대화 화면 |
| **잘 모르겠어요** | "괜찮아요. 좀 더 써보면 보여요." → 홈으로 | 홈 |

### 스펙

| 항목 | 값 |
|------|-----|
| 에센스 한 줄 | Noto Serif KR 28px/500, `#3D3632`, 중앙정렬 |
| 6축 라벨 | Pretendard 12px/400, `#8A8480` |
| 6축 설명 | Pretendard 14px/400, `#3D3632` |
| 6축 바 | 높이 4px, 라운드 2px, 색상: 팔레트 primary |
| 팔레트 원 | 24px, 간격 12px, 중앙 정렬 |
| 가설 질문 | Pretendard 16px/300, `#5A5550`, 이탤릭 |
| 반응 버튼 | Pretendard 14px/400, `#3D3632`, border 1px `#E8E4E0`, 라운드 20px, 패딩 10px 20px |
| 공유 버튼 | Pretendard 13px/400, `#8A8480`, 밑줄, 배경 없음 |
| 구분선 | `#E8E4E0`, 1px |

### 성공 기준

| 지표 | 목표 |
|------|------|
| 에센스 화면 도달율 (로딩 완주) | 95%+ |
| "맞아요" 선택 비율 | 40%+ |
| "좀 다른데" (= 대화 진입) | 30%+ |
| 이탈 (뒤로가기) | 15% 미만 |
| 에센스 → 가설질문까지 스크롤 | 80%+ |

## F4. 공유 — "이거 봐봐"

> 공유 카드가 다음 유저의 F1이 된다.

### 유저 행동 시퀀스

```
1. "맞아요" 탭 후 공유 CTA 노출
2. "카카오톡으로 공유" 탭
3. 공유 카드 생성 (서버) → 카카오 공유 SDK 호출
4. 친구에게 전송
5. 친구가 카드 탭 → 랜딩 (F1-A 경로)
```

### API 호출

```
POST /api/share
{
  "essence": { ... },
  "format": "kakao"  // 또는 "instagram"
}
→ { "imageUrl": "...", "kakaoShare": { ... } }
```

### 공유 카드 레이아웃

**카카오톡** (1:1, 720x720):
```
[배경: 팔레트 가장 밝은 색의 10% opacity]

        "손 닿는 곳의 아늑함을
         모으는 사람"

          ● ● ● ● ●

        droppi →
```

**인스타 스토리** (9:16, 1080x1920):
```
[상단 1/3: 업로드 사진 중 3장 콜라주]

[중앙]
        "손 닿는 곳의 아늑함을
         모으는 사람"

          ● ● ● ● ●

[하단]
        droppi
        나도 해보기 →
```

### 성공 기준

| 지표 | 목표 |
|------|------|
| 에센스 화면에서 공유 버튼 클릭 | 30%+ |
| 공유 카드 → 랜딩 유입 (클릭률) | 15%+ |
| 공유 경로 중 카카오 비율 | 측정 |
| 공유 유입 → 온보딩 완료 전환 | 60%+ |

### 바이럴 루프 정리

```
유저A 에센스 → 공유 → 유저B 유입 → 유저B 에센스 → 공유 → 유저C ...
  ↑                                                         │
  └─────────────────────────────────────────────────────────┘
```

**K-factor 구성요소:**
- i (초대 수) = 에센스 화면에서 공유하는 비율
- c (전환율) = 공유 카드 본 사람 중 온보딩 완료 비율
- **목표**: K > 0.3 (자연 성장 보조)

## F5. 재방문 — "오늘은 이거 가져왔어"

### 진입 경로

| 경로 | 빈도 | 트리거 |
|------|------|--------|
| 텔레그램 봇에 직접 입력 | 높음 | 글 읽다가, 사진 찍다가, 생각나서 |
| 웹앱 홈 | 중간 | 발견 확인하러 |
| 주간 리포트 알림 | 주 1회 | "이번 주엔 '경계' 근처에 있었어요" |

### 화면: 홈 (`/home`)

```
[상단 고정]
  "손 닿는 곳의 아늑함을 모으는 사람"
  (에센스 한 줄, 항상 보임)

[입력 영역]
  ┌─────────────────────────────┐
  │ 오늘은 뭘 가져왔어요?        │
  │                             │
  │  📷  🔗           [보내기]  │
  └─────────────────────────────┘

[최근 발견]
  ┌─────────────────────────────┐
  │ "만드는 건 잘하는데          │
  │  닫는 게 안 돼"             │
  │                             │
  │ #Closer  #통제   3일 전     │
  └─────────────────────────────┘

  ┌─────────────────────────────┐
  │ "넓은 곳이 필요한 상태"      │
  │                             │
  │ #공간  #상태     1주 전      │
  └─────────────────────────────┘

[하단 네비게이션]
  홈    대화    프로필
```

### 입력 → 라우팅

| 유저 입력 | 시스템 동작 |
|-----------|-----------|
| 사진 첨부 + 보내기 | → `/chat` 새 세션, type: image |
| 텍스트 입력 + 보내기 | → `/chat` 새 세션, type: memo |
| URL 포함 텍스트 | → `/chat` 새 세션, type: link |
| 발견 카드 탭 | → `/chat/:sessionId` (해당 세션) |

### 플레이스홀더 로테이션

세션 수에 따라 플레이스홀더가 바뀐다:

| 세션 수 | 플레이스홀더 |
|---------|------------|
| 0~2 | "오늘은 뭘 가져왔어요?" |
| 3~9 | "요즘 저장만 해둔 글 있어요?" |
| 10~19 | "최근에 뭐가 눈에 밟혀요?" |
| 20+ | "오늘은 어디쯤이에요?" |

### 스펙

| 항목 | 값 |
|------|-----|
| 에센스 고정 | Noto Serif KR 18px/500, `#3D3632`, 상단 패딩 20px |
| 입력 영역 배경 | `#F9F8F7`, 라운드 20px, 패딩 12px 16px |
| 입력 텍스트 | Pretendard 15px/400, `#3D3632` |
| 플레이스홀더 | Pretendard 15px/300, `#C4C0BC` |
| 발견 카드 | 배경 `#FFFFFF`, border 1px `#F0EEEC`, 패딩 16px, 라운드 0px |
| 발견 텍스트 | Pretendard 15px/400, `#3D3632` |
| 태그 | Pretendard 12px/400, `#8A8480`, 배경 `#F5F3F0`, 라운드 4px |
| 네비게이션 | 3탭, Pretendard 12px/400, 활성 `#3D3632`, 비활성 `#C4C0BC` |

### 성공 기준

| 지표 | 목표 |
|------|------|
| Day 1 재방문 | 40%+ |
| Day 7 재방문 | 20%+ |
| 홈에서 입력 시작 | 방문당 60%+ |
| 입력 → 대화 전환 | 90%+ |

## F6. 대화 — "아, 그래서 그랬구나"

> 핵심 루프. 입력 → 질문 → 대답 → 질문 → 발견.

### 유저 행동 시퀀스 (사진 입력 예시)

```
1. 홈에서 사진 1장 첨부 → 보내기
2. 대화 화면 전환. 사진이 유저 버블로 표시
3. AI 첫 질문: "이 사진에서 뭐가 끌려요?"         (3초 이내)
4. "빛이 들어오는 방향이요"
5. AI: "빛의 방향을 보는 거네요. 혹시 사진을 찍을 때도
      빛이 어디서 오는지를 먼저 보는 편이에요?"
6. "네, 항상 역광을 찾아요"
7. AI: "에센스에서 '불투명. 빛을 품는 물질들'이었잖아요.
      빛 자체보다 빛을 머금은 것에 끌리는 건지,
      빛이 만드는 경계에 끌리는 건지 — 어느 쪽이에요?"
8. "아... 경계인 것 같아요. 빛과 그림자의 경계"
9. AI 발견 감지:
   💡 "빛 자체가 아니라, 빛이 만드는 경계에 끌린다"
   기록해둘까요?
10. [기록] 탭 → InsightLog 저장
```

### 유저 행동 시퀀스 (글/링크 입력 예시)

```

      불편함을 '견디는' 건지 '선택하는' 건지 —
      어느 쪽이에요?"
5. "선택하는 쪽이요. 일부러 어려운 걸 고르는 편이에요"
6. AI: "에센스에서 '거친 표면 아래'를 찾는다고 했잖아요.
      쉬운 길을 놔두고 거친 쪽을 고르는 건
      디자인 철학인가요, 삶의 태도인가요?"
7. "삶의 태도에 더 가까운 것 같아요"
8. AI 발견 감지:
   💡 "편안함보다 성장의 마찰을 선택하는 패턴"
   기록해둘까요?
```

### API 호출 흐름

```
유저 입력
  → POST /api/chat
    {
      input: { type, content },
      essence: { ... },
      history: [ 이전 메시지들 ]
    }
  ← { message, type: "question" | "insight", insight }

insight가 있으면 → 인사이트 카드 렌더링
유저가 "기록" 탭 → localStorage에 InsightLog 추가
```

### 대화 규칙 (개발 체크리스트)

- [ ] 첫 질문은 입력 후 3초 이내
- [ ] AI는 한 번에 질문 1개만
- [ ] AI는 절대 글을 요약하지 않음
- [ ] AI는 "좋은 글이네요" 같은 감상 금지
- [ ] 에센스 연결은 자연스러울 때만 (매 턴 언급 X)
- [ ] 발견 시 유저의 말 그대로 인용 (AI 언어로 바꾸지 않음)
- [ ] "기록해둘까요?" — 유저 선택 존중

## F7. 축적 — "나 변했네"

> 1회는 대체 가능. 30회는 불가능.

### 축적이 보이는 3가지

| 뷰 | 무엇을 보여주나 | 유저 감정 |
|----|---------------|----------|
| **키워드 추이** | 반복되는 단어의 변화 | "요즘 이 단어가 많네" |
| **타임라인** | 에센스 변화 + 돌파구 | "그때랑 지금이 다르네" |

### 화면: 프로필 (`/profile`)

```
[상단]
  "손 닿는 곳의 아늑함을 모으는 사람"
  2026.02.20 ~

[키워드]
  #완성 ████  (8회)
  #혼자 ███   (5회)
  #속도 ██    (3회)
  #경계 █     (2회, 최근 급등 ↑)

[타임라인]
  현재 ─●─ 에센스 변화: "경계를 그리되..."
       │
       ●─ 돌파구: "속도를 늦추기 → 꺼내는 속도 올리기"
       │
       ●─ 패턴: "통제에 대한 글에서 반복 반응"
       │
  시작 ─●─ 첫 에센스: "손 닿는 곳의 아늑함을..."

[발견 전체 목록]
  (날짜순, 최신 먼저)
```

### 주간 리포트 (텔레그램 / 웹 알림)

**조건**: 해당 주 2세션 이상 시 주말에 발송

```
"이번 주엔 '경계' 주변을 맴돌았어요.
 지난달엔 '아늑함'이 많았는데, 요즘은 경계 쪽으로 가고 있네요."

```

**톤**: 관찰. 느낌표 금지.

### 축적 엔진 처리 (백그라운드)

| 트리거 | 처리 |
|--------|------|
| 인사이트 저장 시 | 키워드 추출 → 기존 클러스터에 매칭 or 신규 생성 |
| 사진 재업로드 시 | 에센스 재분석 → essenceHistory에 추가 → 변화 감지 |
| 주간 (크론) | 키워드 추이 계산 → 리포트 생성 |

### 에센스 변화 감지

유저가 새 사진 5장을 업로드하면 (`/profile` > "에센스 다시 읽기"):

```
기존 에센스와 비교
→ 6축 중 2축 이상 방향 변화 시 "변화" 이벤트
→ 타임라인에 before/after 노드 추가
→ "에센스가 바뀌었어요." 알림
```

### 성공 기준

| 지표 | 목표 |
|------|------|
| 30세션 달성 유저의 에센스 변화 횟수 | 1회+ |
| 프로필 페이지 주간 방문 | 세션 유저의 50%+ |
| 주간 리포트 오픈율 | 40%+ |
| "에센스 다시 읽기" 사용률 | 월 1회+ (10세션 이상 유저) |

## 텔레그램 봇 — 웹앱 병행 시나리오

> 텔레그램은 입력 채널, 웹은 축적 채널. 데이터는 하나.

### 텔레그램 시나리오

```
[첫 메시지 — 에센스 없는 유저]
봇: 사진 5장 보내주세요. 끌리는 것으로. 예쁠 필요 없어요.

[사진 5장 수신 완료]
봇: 다 읽었어요. (에센스 텍스트 결과)
    전체 분석은 여기서 볼 수 있어요 → [웹 링크]

[이후 일상 대화]
유저: (아티클 링크 전송)
봇: 이 글에서 뭐가 걸렸어요?
유저: "..."
봇: (메타대화 질문)
...
봇: 💡 발견: "..."
    기록해둘까요?
유저: 응
봇: 기록했어요. → [웹 링크]

[주간 리포트]
봇: "이번 주엔 '경계' 주변을 맴돌았어요."
    자세히 보기 → [웹 링크]
```

### 채널 간 데이터 동기화

| 데이터 | 저장 위치 | 동기화 |
|--------|---------|--------|
| MetaProfile | localStorage (웹) | 텔레그램 세션 → 웹에 반영 필요 |
| 세션 히스토리 | localStorage (웹) | 텔레그램 대화 → 웹에서 열람 가능 |
| 에센스 | localStorage (웹) | 한 곳에서 수정 → 양쪽 반영 |

**해커톤 단계**: 텔레그램 uid ↔ 웹 localStorage 매핑은 "웹에서 텔레그램 연결" 버튼으로 처리. 서버 DB 없이 웹 세션 토큰으로 연결.
