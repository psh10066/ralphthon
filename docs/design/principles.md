# droppi — 디자인 원칙

drop + I. 사진, 글, 메모를 떨어뜨리면 쌓이고, AI가 취향 에센스를 읽고 메타대화로 자기 이해를 돕는 서비스.

---

## 디자인 참조 우선순위

<HARD-RULE>
구현할 때 디자인 판단이 필요하면 아래 순서로 참조한다:

1. **레퍼런스 이미지** (`docs/design/references/`) — 디자인의 기준. 톤, 여백, 밀도, 타이포, 컬러 사용량을 여기서 읽는다
2. **디자인 원칙** (이 문서) — 왜 그런 톤인지, 어떤 규칙이 있는지
3. **목업** (`mockup/`) — 구조와 흐름의 참고. 디자인 퀄리티의 기준이 아님

목업은 구조를 빠르게 잡기 위한 것이고, 실제 디자인 톤은 레퍼런스 이미지를 따른다.
목업의 컬러, 여백, 폰트 크기를 그대로 복사하지 않는다.
레퍼런스 이미지의 느낌 — 절제, 여백, 텍스트 중심, 거의 무색 — 이 기준이다.
</HARD-RULE>

### 레퍼런스에서 읽어야 할 것

- **여백의 양**: 콘텐츠보다 여백이 많다. 빽빽하면 틀린 것
- **컬러 사용량**: 거의 없다. Warm White + Black + Gray가 95%
- **타이포 위계**: 에센스 한 줄만 크고 세리프. 나머지는 작고 가벼움
- **구조선**: 1px, #040000 계열, 기능적. 장식 아님
- **CTA**: "drop"은 밑줄 텍스트. 화려한 버튼이 아님 (단, 랜딩의 메인 CTA는 블랙 pill)
- **상태 표현**: 뮤트한 텍스트로. 아이콘이나 배지 없음

## 핵심 메타포: 빈 캔버스

서비스는 빈 공간. 사용자에게 영향을 주지 않는다. 저장하고, 들여다보는 장소.
코튼 캔버스 질감. `#EFEEE9` 배경에 블랙 라인 프레임.

## 브랜드 톤

droppi는 캐릭터가 아니라 개념. drop + I(나). 모던하게 담는다.

- 귀엽지 않다. 마스코트, 둥글둥글한 서체, 말랑한 느낌 없음
- 타이포 중심. 직선적. 군더더기 없는
- drop은 중력. 떨어져서 스며드는 것
- 사용자는 dropper. 내부 용어

### 용어 정의

| 용어 | 의미 |
|------|------|
| **drop** | 사진, 글, 메모를 하나 떨어뜨리는 행위. 서비스의 기본 단위 |
| **dropper** | 사용자. drop하는 사람 |
| **에센스** | drop에서 읽어낸 취향의 본질. 한 문장 |
| **켜** | drop이 쌓여 만들어진 층위. 축적의 단위 |

## 디자인 원칙

### 1. 투명하게 (Transparent)

UI 존재감을 줄인다. 콘텐츠가 보인다.

- 컨테이너, 보더, 장식 최소화
- 텍스트와 여백으로 구성

### 2. 여백이 언어다 (Whitespace is Language)

여백은 호흡.

- 에센스 한 줄이 화면 중심. 주변은 비운다
- 정보 밀도 낮게
- 한 번에 하나만

### 3. 콘텐츠가 주인공이다 (Content is the Main Character)

사용자가 넣은 것이 화면을 구성한다.

- 별도의 일러스트, 아이콘, 그래픽 장식 없음
- 사용자의 사진, 텍스트, 에센스가 화면에 있는 전부
- 서비스 고유의 시각 요소는 최소한으로

### 4. 한 번에 하나 (One Thing at a Time)

퍼널의 각 단계에서 사용자가 해야 할 일은 하나.

```
"사진 올려" → "읽고 있어" → "이게 당신이야" → "어떻게 생각해?" → "글 가져와봐"
```

- 선택지를 줄인다
- 경로를 명확히 한다

### 5. 시적이되 모호하지 않게 (Poetic but Precise)

에센스 언어는 감각적이지만 구체적이다. 단, 과잉되지 않게.

| 안 되는 것 | 좋은 것 |
|-----------|--------|
| "따뜻한 사람" | "손 닿는 곳의 아늑함을 모으는 사람" |
| "당신의 영혼이 빛나는 순간" | 오글거림. 하지 않는다 |

설명하지 말고 보여준다. 시적인 건 좋지만 민망하면 실패.

## 컬러 시스템

### 브랜드 컬러 팔레트

BI에서 정의된 공식 컬러.

| 이름 | HEX | 분류 |
|------|-----|------|
| **Warm White** | `#EFEEE9` | 기본 배경 |
| **White** | `#FFFFFF` | 카드/입력 필드 배경 |
| **Black** | `#040000` | 텍스트, 구조선, 로고 |
| **Dark Gray** | `#45525A` | 보조 텍스트 |
| **Gray** | `#707980` | 비활성, 플레이스홀더 |
| **Brand Blue** | `#7A9CB1` | 브랜드 메인 |
| **Light Blue** | `#A5B7C5` | 브랜드 보조 |
| **Olive Green** | `#C2C9A6` | 엑센트 |
| **Light Mint** | `#CFE2CF` | 엑센트 |

### 일상 UI = 3색만

화면의 95%는 이 3색으로 구성된다. 레퍼런스처럼 절제된 톤.

| 용도 | 컬러 | HEX |
|------|------|-----|
| 배경 | Warm White | `#EFEEE9` |
| 텍스트 | Black | `#040000` |
| 뮤트 텍스트 (보조, 날짜, 라벨) | Gray | `#707980` |

이 3색 외의 컬러는 **의미가 있을 때만** 등장한다.

### 컬러가 등장하는 순간

컬러는 장식이 아니라 **의미의 신호**. 평소에는 보이지 않다가, 의미 있는 순간에만 나타난다.

| 순간 | 컬러 | 이유 |
|------|------|------|
| 발견이 생겼을 때 | Olive Green `#C2C9A6` | 인사이트 블록 배경. "뭔가 읽혔다"는 신호 |
| 에센스가 바뀌었을 때 | Brand Blue `#7A9CB1` | 변화 하이라이트. 드문 순간이라 눈에 띄어야 함 |
| 월간 흐름에서 지배적 주제 | Light Blue `#A5B7C5` | 프로필 흐름 차트의 강조 |
| 공유 이미지 | Light Mint `#CFE2CF` | 공유 카드 배경. 외부에서 브랜드를 인식하는 접점 |

**등장하지 않는 곳:**
- 일반 카드 배경 — Warm White 그대로
- 태그 칩 — 테두리만 (Gray `#707980` 1px border), 배경은 투명
- 버튼 — Black 텍스트 + 밑줄. 배경색 버튼 없음 (레퍼런스의 "drop" 참고)
- 입력 필드 — White `#FFFFFF` 배경 + Gray 구조선
- 하단 탭 — Black (활성) / Gray (비활성). 컬러 없음

### BI 에셋

`public/brand/` 에 SVG 파일 보관. 3가지 컬러 x 3가지 형태.

| 형태 | Black | White | Brand Blue |
|------|-------|-------|------------|
| 풀 (심볼+워드마크) | `logo-full-black.svg` | `logo-full-white.svg` | `logo-full-blue.svg` |
| 심볼만 | `logo-symbol-black.svg` | `logo-symbol-white.svg` | `logo-symbol-blue.svg` |
| 워드마크만 | `logo-wordmark-black.svg` | `logo-wordmark-white.svg` | `logo-wordmark-blue.svg` |

- 심볼: 물방울 2개 (위아래 대칭) + 하단 곡선. drop의 시각화
- 워드마크: "droppi" 커스텀 레터링
- 일상 UI에서는 Black 로고만 사용 (Warm White 배경 위)
- Brand Blue 로고는 공유 이미지, 외부 노출용

## 태그 컬러 원칙

태그는 유일하게 브랜드 컬러가 일상적으로 등장하는 곳이다.
단, 은은하게 — 배경 opacity 10-25%, 테두리 opacity 20-40%.

주제 태그는 주제별로 고유 컬러가 있다.
스타일 태그는 주제보다 더 연하고 이탤릭.

컬러가 너무 눈에 띄면 안 된다. 태그가 10개 나열됐을 때 산만하지 않아야 한다.
"있는 듯 없는 듯" — 자세히 보면 다르지만, 전체 톤은 Warm White + Gray 안에 머문다.

### 컬러 안티패턴

- 태그에 배경색 넣기 — 태그가 많아지면 화면이 산만해짐. 테두리만
- 버튼에 Brand Blue 배경 — 레퍼런스처럼 텍스트+밑줄이 맞음
- 카드마다 다른 배경색 — 전부 Warm White. 구분은 1px 라인으로
- 섹션마다 배경색 교차 — 안 함. 전체가 Warm White

## 레이아웃 원칙

### 타이포 중심 (Typography-driven)

텍스트가 화면을 이끈다. 텍스트와 이미지가 잘 보이는 것이 최우선.

- 큰 타이포그래피로 위계를 명확히 한다
- 제목은 크고 굵게. 본문과의 크기 차이를 충분히 둔다
- 사용자가 넣은 사진/이미지는 넉넉한 크기로 보여준다. 썸네일로 축소하지 않는다

### 컬러 블록 그룹핑

섹션 구분을 라인이 아니라 배경색 블록으로 한다.

- 카드/블록의 배경색이 그룹핑 역할을 한다
- 한 화면에 2~3개 블록까지. 더 많으면 산만해진다
- 블록 안에서는 미색 배경 + 블랙 텍스트로 통일

### 리스트 구분

- 가는 라인(1~1.5px)으로 항목을 구분한다
- 라인은 기능적. 장식이 아니라 구조

## 타이포그래피

| 요소 | 폰트 | 사이즈 | 웨이트 | 행간 |
|------|------|--------|--------|------|
| 에센스 한 줄 | Noto Serif KR | 28-36px | 400-500 | 1.7 |
| 본문 | Pretendard | 14-16px | 300-400 | 1.7-1.8 |

에센스는 세리프로 무게감을 주고, 본문은 산세리프로 가볍게.

## 반응형 기준

모바일 퍼스트. 데스크톱은 1440px 뷰포트 기준 1200px를 채우는 형태.

### 레이아웃

| 기준 | 모바일 (< 768px) | 태블릿 (768~1023px) | 데스크톱 (≥ 1024px) |
|------|-----------------|-------------------|-------------------|
| 콘텐츠 너비 | 100% | 100% | **max-width: 1200px** |
| 좌우 패딩 | 20px | 32px | 40px |
| 그리드 | 1열 | 2열 | 2~3열 |
| 정렬 | - | - | 센터 (좌우 여백 자동) |
| 터치 타겟 | 최소 44px | 44px | - |

- **fullwidth 금지.** 콘텐츠가 1200px를 넘어 화면 끝까지 가지 않는다
- 375px에서 깨지지 않는 게 최우선
- 가로 스크롤 금지 (예시 카드 섹션만 예외)
- 텍스트 한 줄이 80자를 넘지 않도록 (데스크톱에서 max-width 제한)
- 텍스트는 최소 14px. 에센스 한 줄은 22~28px (반응형)

## 시간 흐름 UI 원칙

시간에 따른 변화를 보여주는 UI. 축적이 곧 해자.

### 타임라인

- 세로 스크롤 타임라인. 위가 현재, 아래가 과거
- 각 노드: 날짜 + 에센스 한 줄 또는 인사이트 한 줄
- 노드 사이 연결선은 가늘고(1~1.5px) 투명하게. 주인공은 노드의 텍스트

### Before/After 비교

- "6개월 전의 나 vs 지금의 나" 같은 비교를 자연스럽게 보여준다
- 두 시점의 에센스를 나란히 놓되, 변화를 강조하지 않는다. 차이는 사용자가 느끼는 것
- 에센스 변화를 시각적으로 보여줄 때도 숫자가 아니라 톤의 변화로

### 변화 감지 표현

| 표현 | 방식 |
|------|------|
| 에센스 변화 | "6개월 전엔 미니멀했는데 요즘은 따뜻한 쪽으로" — 서술형 |
| 패턴 발견 | "'통제'에 대한 글에서 반복 반응" — 태그 하이라이트 |
| 돌파구 | 전환 사례를 카드로 강조 (before 문장 → after 문장) |

### 안티패턴

- 그래프로 성장을 보여주지 않는다 (자기 이해는 성장이 아니라 발견)
- "n% 변화" 같은 정량화 금지
- 시간순 나열만 하지 않는다. 패턴과 전환점을 묶어서 보여준다

## 텔레그램 톤

텔레그램 봇이 첫 접점이다. 웹앱과 동일한 톤을 유지한다.

### 원칙

- 이모지 최소. 쓸 때는 문장 끝에 하나만
- 질문 중심. 정보를 주지 않고 물어본다
- 푸시 메시지에 느낌표 금지

### 톤 예시

| 안 되는 것 | 좋은 것 |
|-----------|--------|
| "새로운 에센스가 발견되었습니다!" | "사진에서 뭔가 읽혔어요. 볼래요?" |
| "분석 완료! 결과를 확인하세요" | "다 읽었어요." |
| "오늘의 인사이트를 확인해보세요" | "어제 가져온 글, 하나 물어봐도 돼요?" |

### 메시지 구조

- 한 메시지에 한 가지만. 복합 메시지 금지
- 버튼은 최대 2개. "볼래요" / "나중에"
- 긴 결과는 웹앱으로 연결. 텔레그램 안에서 다 보여주지 않는다

## 인터랙션

### 기본

- **전환**: 부드럽고 천천히. 급하지 않게
- **로딩**: 단계를 보여주되 차분하게. "읽고 있어요" 같은 톤
- **호버/클릭**: 미세한 변화만. 과한 피드백 없음

### drop 동기화 모션

다른 채널에서 콘텐츠가 넘어올 때:

1. 상단에서 물방울 형태가 떨어짐
2. 미색 표면에 착지
3. 흡수되며 리스트에 자리 잡음

- 여러 개면 하나씩 순차적으로
- 바운스 없음. 중력 + 흡수
- 모션 전체 0.6~0.8초

## 안티패턴
