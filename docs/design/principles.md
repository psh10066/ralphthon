# droppi — 디자인 원칙

drop + I. 사진, 글, 메모를 떨어뜨리면 쌓이고, AI가 취향 에센스를 읽고 메타대화로 자기 이해를 돕는 서비스.

---

## 핵심 메타포: 빈 캔버스

서비스는 빈 공간. 사용자에게 영향을 주지 않는다. 저장하고, 들여다보는 장소.
코튼 캔버스 질감. 미색 배경에 블랙 라인 프레임.

## 브랜드 톤

droppi는 캐릭터가 아니라 개념. drop + I(나). 모던하게 담는다.

- 귀엽지 않다. 마스코트, 둥글둥글한 서체, 말랑한 느낌 없음
- 타이포 중심. 직선적. 군더더기 없는
- drop은 중력. 떨어져서 스며드는 것
- 사용자는 dropper. 내부 용어

### 용어 정의

| 용어 | 의미 |
|------|------|
| **drop** | 사진, 글, 메모를 하나 떨어뜨리는 행위. 서비스의 기본 단위 |
| **dropper** | 사용자. drop하는 사람 |
| **에센스** | drop에서 읽어낸 취향의 본질. 한 문장 |
| **켜** | drop이 쌓여 만들어진 층위. 축적의 단위 |

## 디자인 원칙

### 1. 투명하게 (Transparent)

UI 존재감을 줄인다. 콘텐츠가 보인다.

- 컨테이너, 보더, 장식 최소화
- 텍스트와 여백으로 구성

### 2. 여백이 언어다 (Whitespace is Language)

여백은 호흡.

- 에센스 한 줄이 화면 중심. 주변은 비운다
- 정보 밀도 낮게
- 한 번에 하나만

### 3. 콘텐츠가 주인공이다 (Content is the Main Character)

사용자가 넣은 것이 화면을 구성한다.

- 별도의 일러스트, 아이콘, 그래픽 장식 없음
- 사용자의 사진, 텍스트, 에센스가 화면에 있는 전부
- 서비스 고유의 시각 요소는 최소한으로

### 4. 한 번에 하나 (One Thing at a Time)

퍼널의 각 단계에서 사용자가 해야 할 일은 하나.

```
"사진 올려" → "읽고 있어" → "이게 당신이야" → "어떻게 생각해?" → "글 가져와봐"
```

- 선택지를 줄인다
- 경로를 명확히 한다

### 5. 시적이되 모호하지 않게 (Poetic but Precise)

에센스 언어는 감각적이지만 구체적이다. 단, 과잉되지 않게.

| 안 되는 것 | 좋은 것 |
|-----------|--------|
| "따뜻한 사람" | "손 닿는 곳의 아늑함을 모으는 사람" |
| "당신의 영혼이 빛나는 순간" | 오글거림. 하지 않는다 |

설명하지 말고 보여준다. 시적인 건 좋지만 민망하면 실패.

## 컬러 시스템

### 기본 팔레트

| 용도 | 값 | 비고 |
|------|-----|------|
| 배경 | 미색 (warm off-white) | 순백이 아닌 약간의 온기 |
| 텍스트 | 블랙~다크 | 명확한 가독성 |
| 구조선 | 블랙, 1~1.5px | 기능적 구분. 가볍지만 명확한 프레임 |
| 카드/블록 배경 | 연한 뮤트 톤 | 차분하되 파스텔은 아닌. 그룹핑에 사용 |
| 강조 악센트 | 형광 계열 (살짝만) | 발견/변화/전환 같은 특별한 순간에만 |

### 컬러 톤 원칙

- **차분한 톤**: 뮤트. 파스텔 아님. 힘이 있는 톤
- **형광은 양념**: 전체를 지배하지 않는다. 발견/변화 순간에만
- **컬러 = 맥락**: 장식이 아니라 상태 표시

### 컬러 등장 규칙

| 맥락 | 컬러 | 의미 |
|------|------|------|
| 일상 (입력/대화) | 미색 + 블랙 | 기본 상태 |
| 첫 발견 (에센스 생성) | 연한 뮤트 톤 블록 | 에센스 생성됨 |
| 고도화 (패턴, 연결) | 조금 더 선명한 톤 | 패턴 감지 |
| 변화 감지 (전환점) | 형광 악센트 | 전환점 |
| 리포트 | 컬러 조합 | 축적 정리 |

## 레이아웃 원칙

### 타이포 중심 (Typography-driven)

텍스트가 화면을 이끈다. 텍스트와 이미지가 잘 보이는 것이 최우선.

- 큰 타이포그래피로 위계를 명확히 한다
- 제목은 크고 굵게. 본문과의 크기 차이를 충분히 둔다
- 사용자가 넣은 사진/이미지는 넉넉한 크기로 보여준다. 썸네일로 축소하지 않는다

### 컬러 블록 그룹핑

섹션 구분을 라인이 아니라 배경색 블록으로 한다.

- 카드/블록의 배경색이 그룹핑 역할을 한다
- 한 화면에 2~3개 블록까지. 더 많으면 산만해진다
- 블록 안에서는 미색 배경 + 블랙 텍스트로 통일

### 리스트 구분

- 가는 라인(1~1.5px)으로 항목을 구분한다
- 라인은 기능적. 장식이 아니라 구조

## 타이포그래피

| 요소 | 폰트 | 사이즈 | 웨이트 | 행간 |
|------|------|--------|--------|------|
| 에센스 한 줄 | Noto Serif KR | 28-36px | 400-500 | 1.7 |
| 본문 | Pretendard | 14-16px | 300-400 | 1.7-1.8 |

에센스는 세리프로 무게감을 주고, 본문은 산세리프로 가볍게.

## 모바일 반응형 기준

텔레그램 웹앱이 주 플랫폼이므로 모바일 퍼스트로 설계한다.

| 기준 | 값 |
|------|-----|
| 기본 뷰포트 | 375px |
| 최대 너비 | 480px (`max-width`) |
| 사이드 패딩 | 20px |
| 터치 타겟 | 최소 44px |

- 375px에서 깨지지 않는 게 최우선. 480px 초과 시 중앙 정렬 + 좌우 여백
- 텔레그램 웹앱 헤더(약 56px)를 감안한 safe area 확보
- 가로 스크롤 절대 금지. 6축 분석도 세로 스택으로 배치
- 텍스트는 최소 14px. 에센스 한 줄은 24-28px (모바일 기준)

## 지형도 시각화 원칙

에센스가 쌓이면 "취향 지형도"가 만들어진다. 등고선 스타일로 자기 취향의 지형을 보여준다.

### 시각 스타일

- **등고선 맵**: 유기적 곡선. 격자나 차트가 아니라 지형도
- **색상**: 미색~블랙 그라데이션으로 밀도 표현. 리포트에서만 컬러 사용
  - 높은 밀도 = 진한 톤
  - 낮은 밀도 = 옅은 톤
  - 배경 = 미색
- **등고선 간격**: 3-5단계
- **라벨**: 클러스터 중심에 한 단어 ("아늑함", "통제", "질감")
- **라인**: 1~1.5px. 블랙

### 인터랙션

- 클러스터 탭 → 해당 에센스/인사이트 목록 펼침
- 핀치 줌 없음. 전체 지형이 한 화면에 들어와야 한다
- 처음엔 섬 하나. 세션이 쌓이면 대륙이 된다

### 안티패턴

- 레이더 차트, 막대 그래프, 파이 차트 금지
- 숫자 라벨 금지 (점수화하지 않는다)
- 3D 효과 금지

## 시간 흐름 UI 원칙

시간에 따른 변화를 보여주는 UI. 축적이 곧 해자.

### 타임라인

- 세로 스크롤 타임라인. 위가 현재, 아래가 과거
- 각 노드: 날짜 + 에센스 한 줄 또는 인사이트 한 줄
- 노드 사이 연결선은 가늘고(1~1.5px) 투명하게. 주인공은 노드의 텍스트

### Before/After 비교

- "6개월 전의 나 vs 지금의 나" 같은 비교를 자연스럽게 보여준다
- 두 시점의 에센스를 나란히 놓되, 변화를 강조하지 않는다. 차이는 사용자가 느끼는 것
- 6축 값의 이동을 시각적으로 보여줄 때도 숫자가 아니라 톤의 변화로

### 변화 감지 표현

| 표현 | 방식 |
|------|------|
| 에센스 변화 | "6개월 전엔 미니멀했는데 요즘은 따뜻한 쪽으로" — 서술형 |
| 패턴 발견 | "'통제'에 대한 글에서 반복 반응" — 태그 하이라이트 |
| 돌파구 | 전환 사례를 카드로 강조 (before 문장 → after 문장) |

### 안티패턴

- 그래프로 성장을 보여주지 않는다 (자기 이해는 성장이 아니라 발견)
- "n% 변화" 같은 정량화 금지
- 시간순 나열만 하지 않는다. 패턴과 전환점을 묶어서 보여준다

## 텔레그램 톤

텔레그램 봇이 첫 접점이다. 웹앱과 동일한 톤을 유지한다.

### 원칙

- 이모지 최소. 쓸 때는 문장 끝에 하나만
- 질문 중심. 정보를 주지 않고 물어본다
- 푸시 메시지에 느낌표 금지

### 톤 예시

| 안 되는 것 | 좋은 것 |
|-----------|--------|
| "새로운 에센스가 발견되었습니다!" | "사진에서 뭔가 읽혔어요. 볼래요?" |
| "분석 완료! 결과를 확인하세요" | "다 읽었어요." |
| "오늘의 인사이트를 확인해보세요" | "어제 가져온 글, 하나 물어봐도 돼요?" |

### 메시지 구조

- 한 메시지에 한 가지만. 복합 메시지 금지
- 버튼은 최대 2개. "볼래요" / "나중에"
- 긴 결과는 웹앱으로 연결. 텔레그램 안에서 다 보여주지 않는다

## 인터랙션

### 기본

- **전환**: 부드럽고 천천히. 급하지 않게
- **로딩**: 단계를 보여주되 차분하게. "읽고 있어요" 같은 톤
- **호버/클릭**: 미세한 변화만. 과한 피드백 없음

### drop 동기화 모션

다른 채널에서 콘텐츠가 넘어올 때:

1. 상단에서 물방울 형태가 떨어짐
2. 미색 표면에 착지
3. 흡수되며 리스트에 자리 잡음

- 여러 개면 하나씩 순차적으로
- 바운스 없음. 중력 + 흡수
- 모션 전체 0.6~0.8초

---

## 안티패턴

하지 않는 것들:

- 둥근 카드에 그림자 잔뜩
- 이모지 과다 사용
- 컬러풀한 버튼
- 대시보드 느낌의 레이아웃
- "AI가 분석했습니다!" 같은 기술 과시
- 오글거리는 감성 카피
- frosted glass, 글로우 효과
- 파스텔 톤 (달콤하고 가벼운 느낌)
- 사진/텍스트를 작게 축소하는 것

---

## 레퍼런스 무드

- 코튼 캔버스 질감. 미색 배경
- 1~1.5px 블랙 라인 프레임
- 타이포 중심. 큰 텍스트 + 넉넉한 이미지
- 뮤트 톤 컬러 블록 그룹핑. 종종 형광 악센트
- 가는 라인 리스트
