# 디자인 검수 체크리스트 — 컨셉 전환 반영 확인

> 와이어프레임/디자인 시안이 도착하면, 아래 항목을 하나씩 대조한다.
> 6가지 방향 전환이 빠짐없이 반영됐는지 확인하는 것이 목적.

## 1. "거울" 컨셉 제거 — 6축이 wow point가 아님

**기존 스펙 충돌 지점:**
- `screens.md:3` — 디자인 톤에 "거울" 명시
- `screens.md:112-150` — "거울 — 에센스 결과" 섹션 (6축 텍스트 리스트 + 컬러 팔레트)
- `funnel.md:14` — `[4] 거울 ← wow point`
- `funnel.md:52-65` — 거울 섹션 카피 (에센스 한 줄 32px, 6축, 팔레트)
- `screens.md:370-404` — 프로필 화면에 6축 전체 표시
- `user-scenario-funnel.md:13` — `F3. WOW (First Drop) → "아, 이게 나야?"`

**검수 항목:**

| # | 확인 사항 | Pass/Fail |
|---|----------|-----------|
| 1 | 6축(양감/질감/투명도/촉각/무게/온도) 비주얼이 완전히 빠졌는가 | |
| 2 | wow point가 다른 것으로 대체됐는가, 아니면 wow 자체를 제거했는가 | |
| 3 | "거울"이라는 단어/메타포가 UI 어디에도 없는가 | |
| 4 | 에센스 결과 화면이 어떤 형태로 바뀌었는가 (대체물 확인) | |
| 5 | 프로필 화면에서 6축 표시가 제거됐는가 | |

## 3. 입력을 사진 5장으로 한정하지 않음 — 뭐든 하나만 drop

**기존 스펙 충돌 지점:**
- `screens.md:18` — 랜딩 카피 "사진 5장이면 당신이 보입니다"
- `screens.md:36-48` — 업로드 화면 (5장 그리드, 5장 미만 비활성)
- `screens.md:55-63` — 업로드 방식 (멀티셀렉트, 5장 초과 제한)
- `funnel.md:10` — `[2] 업로드 → 끌리는 사진 5장`
- `funnel.md:37-43` — 업로드 섹션 (갤러리에서 최대 5장)
- `user-scenario-funnel.md:44` — "사진 5장을 드롭해 보세요"

**검수 항목:**

| # | 확인 사항 | Pass/Fail |
|---|----------|-----------|
| 1 | 랜딩 카피가 "사진 5장"이 아닌 열린 입력으로 바뀌었는가 | |
| 2 | 업로드 UI가 5장 그리드가 아닌 단일 드롭존인가 | |
| 3 | 사진/글/링크 등 입력 타입 구분 없이 하나의 진입점인가 | |
| 4 | 활성화 조건이 5장이 아닌 1개 입력으로 바뀌었는가 | |
| 5 | 랜딩 예시 카드에 사진 외 입력(글, 링크) 예시가 포함됐는가 | |

## 4. 비주얼만 분석하지 않음 — 내용 + 무드, 메타대화 스킬 사용

**기존 스펙 충돌 지점:**
- `dialogue.md:52-54` — photo 전략이 색감/구도/감각 중심
- `screens.md:102-107` — 로딩 4단계 ("색감을 보고 있어요", "질감을 느끼고 있어요")
- `funnel.md:86-91` — 로딩 텍스트 전부 감각 분석 기반

**검수 항목:**

| # | 확인 사항 | Pass/Fail |
|---|----------|-----------|
| 1 | 로딩 텍스트가 감각("색감", "질감")이 아닌 내용+무드 읽기로 바뀌었는가 | |
| 2 | 분석 결과가 6축이 아닌 메타대화 형식(관찰 + 열린 질문)으로 나오는가 | |
| 3 | 글/링크 입력 시에도 동일한 분석 플로우가 적용되는가 | |
| 4 | 사진의 "내용"(뭐가 찍혀있는지)을 읽는 UI가 있는가 | |

## 5. 대화를 강제하지 않음 — 반응하면 대화, 안 하면 쌓임

**기존 스펙 충돌 지점:**
- `screens.md:89-93` — "사용자가 자유 응답을 입력하면 → 즉시 /chat으로 전환"
- `screens.md:153-168` — 에센스→채팅 강제 전환 플로우
- `funnel.md:62-64` — "답하면 즉시 /chat으로 전환"

**검수 항목:**

| # | 확인 사항 | Pass/Fail |
|---|----------|-----------|
| 1 | 분석 결과 화면에서 응답 안 해도 다음으로 넘어갈 수 있는가 | |
| 2 | "쌓임"이 어떻게 표현되는가 — 홈에서 과거 drop 결과를 볼 수 있는가 | |
| 3 | 대화 진입이 선택적인 UI인가 (필수 입력 필드가 아닌 옵셔널) | |
| 4 | 응답하지 않고 닫았을 때의 상태 — 결과가 어디에 저장되는가 | |
| 5 | 나중에 다시 돌아와서 대화를 시작할 수 있는가 | |
