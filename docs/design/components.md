# droppi — UI 컴포넌트

디자인 원칙(`principles.md`)을 따르는 컴포넌트 상세 정의.
블록형. 요소 적게. 텍스트와 이미지가 잘 보이게.

---

## 디자인 토큰 (공통)

| 토큰 | 값 |
|------|-----|
| 배경 (기본) | 미색 (warm off-white) |
| 텍스트 (기본) | 블랙~다크 |
| 구조선 | 블랙, 1~1.5px |
| 블록 배경 | 연한 뮤트 톤 (맥락에 따라 변화) |
| 강조 악센트 | 형광 계열 (발견/변화 순간에만) |
| 카드 라운드 | 8px |
| 카드 그림자 | 없음 |
| 기본 폰트 | Pretendard |
| 에센스 폰트 | Noto Serif KR |

---

## 1. 대화 버블

메타대화 기본 UI.

### 구조

```
사용자 → 오른쪽. 뮤트 톤 블록 배경
AI → 왼쪽. 배경 없음. 텍스트만
```

### 스펙

| 항목 | 사용자 버블 | AI 버블 |
|------|-----------|---------|
| 배경 | 연한 뮤트 톤 블록 | transparent |
| 텍스트 | Pretendard 15px/400 | Pretendard 15px/300 |
| 라운드 | 8px | 없음 |
| 최대 너비 | 280px | 320px |
| 패딩 | 12px 16px | 0px |

### 사진 첨부

- 넉넉하게. 썸네일로 축소하지 않음
- 최대 너비 280px, 비율 유지
- 탭 → 풀스크린

### 상태

- **타이핑 중**: 점 3개 페이드 (0.8초 주기). 차분하게
- **인사이트 감지**: AI 버블 안에 인사이트 블록이 인라인으로 들어감

## 2. 에센스 카드

drop에서 읽어낸 취향 에센스.

### 구조

```
┌─────────────────────────────┐
│                             │
│  "손 닿는 곳의 아늑함을       │
│   모으는 사람"               │
│                             │
│  양감 ━━━━━━━━              │
│  질감 ━━━━━━                │
│  투명 ━━                    │
│  촉각 ━━━━━━━━━             │
│  무게 ━━━━━━                │
│  온도 ━━━━━━━━              │
│                             │
└─────────────────────────────┘
```

- 에센스 한 줄 크게 가운데
- 6축 바 형태, 블랙 1~1.5px
- 팔레트 원 없음

### 상태

| 상태 | 설명 |
|------|------|
| 생성 중 | 로딩 → 텍스트 페이드인 |
| 완성 | 에센스 한 줄 + 6축 |
| 축소 | 리스트에서 — 에센스 한 줄만 |

### 맥락별 컬러

| 맥락 | 카드 배경 |
|------|----------|
| 첫 에센스 | 연한 뮤트 톤 블록 |
| 에센스 고도화 | 조금 더 선명한 뮤트 톤 |
| 에센스 변화 감지 | 형광 악센트 (살짝) |

### 디자인 토큰

| 토큰 | 값 |
|------|-----|
| 에센스 텍스트 | Noto Serif KR 24-28px/500 |
| 6축 라벨 | Pretendard 11px/400, 뮤트 |
| 6축 바 | 블랙, 1~1.5px |
| 카드 패딩 | 24px |
| 카드 라운드 | 8px |

## 3. 인사이트 블록

메타대화에서 발견된 인사이트.

### 구조

```
┌─────────────────────────────┐
│                             │
│  "나는 만드는 건 잘하는데     │
│   닫는 게 안 돼"             │
│                             │
│  #Closer  #통제  #패턴       │
│                             │
└─────────────────────────────┘
```

- 사용자 원문 그대로 표시
- 태그 하단 배치
- 에센스 연결 있으면 하단 한 줄. 없으면 비움

### 상태

| 상태 | 블록 배경 |
|------|----------|
| 신규 | 뮤트 톤 블록 |
| 읽음 | 미색 (기본 배경과 동일) |
| 패턴 연결 | 좌측에 블랙 세로선 1~1.5px |

### 디자인 토큰

| 토큰 | 값 |
|------|-----|
| 발견 텍스트 | Pretendard 16px/400 |
| 태그 | Pretendard 12px/400, 뮤트, 배경 없음 |
| 에센스 연결 | Pretendard 13px/300, 뮤트 |
| 블록 패딩 | 20px |
| 블록 라운드 | 8px |

## 4. 지형도

에센스/인사이트 축적 시각화.

### 시각 스타일

- 등고선 맵. 블랙 라인 1~1.5px
- 배경은 미색. 밀도에 따라 미색~블랙 그라데이션
- 라벨은 클러스터 중심에 한 단어

### 상태

| 상태 | 설명 |
|------|------|
| 초기 (1-5 drop) | 섬 하나 |
| 성장 (6-15 drop) | 2-3개 클러스터 |
| 안정 (16+ drop) | 대륙. 클러스터 간 연결 |

### 인터랙션

- 클러스터 탭 → 해당 에센스/인사이트 목록 (바텀시트)
- 핀치 줌 없음. 한 화면에 전체
- 좌우 스와이프 → 시간별 변화

### 디자인 토큰

| 토큰 | 값 |
|------|-----|
| 등고선 | 블랙, 1~1.5px |
| 클러스터 라벨 | Pretendard 13px/500 |
| 배경 | 미색 |
| 최소 높이 | 320px |

## 5. 타임라인

시간순 변화 표시. 세로 스크롤.

### 구조

가는 세로선(1~1.5px) 위에 노드가 놓인다. 위가 현재, 아래가 과거.

### 노드 유형

| 유형 | 표시 |
|------|------|
| 에센스 변화 | 에센스 한 줄 비교 (before → after) |
| 패턴 발견 | 태그 + 설명 한 줄 |
| 돌파구 | before 문장 → after 문장. 강조 블록 |
| 첫 에센스 | 에센스 카드 축소형 |

### 맥락별 컬러

- 일반 노드: 미색 배경
- 돌파구 노드: 뮤트 톤 블록
- 변화 감지 노드: 형광 악센트 (살짝)

### 디자인 토큰

| 토큰 | 값 |
|------|-----|
| 세로선 | 블랙, 1~1.5px |
| 날짜 | Pretendard 12px/400, 뮤트 |
| 노드 텍스트 | Pretendard 15px/400 |
| before 텍스트 | Pretendard 14px/300, 뮤트, 취소선 |
| after 텍스트 | Pretendard 14px/400 |
| 노드 마커 | 블랙 원 6px (기본), 10px (돌파구) |

## 7. 입력 영역

drop 입력. 사진, 텍스트, 링크.

### 구조

```
┌─────────────────────────────┐
│  텍스트 입력                  │
│  "요즘 저장만 해둔 글 있어요?" │
│                             │
│  📷  🔗            [drop]   │
└─────────────────────────────┘
```

- 플레이스홀더 질문형
- drop 버튼: 블랙, 텍스트 "drop"
- 아이콘 사진/링크 두 개만

### 상태

| 상태 | 설명 |
|------|------|
| 비어있음 | 플레이스홀더 표시 |
| 입력 중 | 자동 높이 확장 (최대 5줄) |
| 사진 첨부 | 입력 위에 프리뷰. 넉넉한 크기 |
| 링크 첨부 | 입력 위에 제목 + 도메인 한 줄 |
| 전송 중 | drop 버튼 → 로딩 |

### 디자인 토큰

| 토큰 | 값 |
|------|-----|
| 입력 배경 | 미색보다 살짝 밝은 톤 |
| 입력 텍스트 | Pretendard 15px/400 |
| 플레이스홀더 | Pretendard 15px/300, 뮤트 |
| 아이콘 | 24px, 블랙 |
| drop 버튼 | Pretendard 14px/500, 블랙 |
| 입력 라운드 | 8px |
| 입력 패딩 | 12px 16px |

## 8. 로딩

에센스 분석 중 표시.

### 구조

```
┌─────────────────────────────┐
│                             │
│  ✓ 보고 있어요               │
│  ✓ 읽고 있어요               │
│  ○ 찾고 있어요               │  ← 현재
│  ○ 거의 다 왔어요            │
│                             │
└─────────────────────────────┘
```

- 화면 중앙. 텍스트만
- 완료 시 에센스 카드로 페이드 전환

### 디자인 토큰

| 토큰 | 값 |
|------|-----|
| 텍스트 | Pretendard 15px/400 |
| 완료 마커 (✓) | 블랙 |
| 대기 마커 (○) | 뮤트 |
| 펄스 | 1.2초 주기 |
| 단계 간격 | 16px |

## 9. Drop 동기화 모션

다른 채널에서 콘텐츠가 넘어올 때.

### 모션 시퀀스

1. 상단에서 물방울 형태 떨어짐
2. 미색 표면 착지
3. 흡수 → 리스트에 배치

### 원칙

- 여러 개면 순차적으로
- 바운스 없음. 중력 + 흡수
- 전체 0.6~0.8초
- 물방울은 추상적 형태. 캐릭터 아님

### 디자인 토큰

| 토큰 | 값 |
|------|-----|
| 물방울 크기 | 12~16px |
| 물방울 색 | 블랙, opacity 0.3 |
| 떨어짐 duration | 0.3초 |
| 흡수 duration | 0.3~0.5초 |
| 연속 drop 간격 | 0.2초 |
