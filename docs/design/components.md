# droppi — UI 컴포넌트

디자인 원칙(`principles.md`)을 따르는 컴포넌트 상세 정의.
블록형. 요소 적게. 텍스트와 이미지가 잘 보이게.

## 1. 대화 버블

메타대화 기본 UI.

### 구조

```
사용자 → 오른쪽. 뮤트 톤 블록 배경
AI → 왼쪽. 배경 없음. 텍스트만
```

### 스펙

| 항목 | 사용자 버블 | AI 버블 |
|------|-----------|---------|
| 배경 | 연한 뮤트 톤 블록 | transparent |
| 텍스트 | Pretendard 15px/400 | Pretendard 15px/300 |
| 라운드 | 8px | 없음 |
| 최대 너비 | 280px | 320px |
| 패딩 | 12px 16px | 0px |

### 사진 첨부

- 넉넉하게. 썸네일로 축소하지 않음
- 최대 너비 280px, 비율 유지
- 탭 → 풀스크린

### 상태

- **타이핑 중**: 점 3개 페이드 (0.8초 주기). 차분하게
- **인사이트 감지**: AI 버블 안에 인사이트 블록이 인라인으로 들어감

## 2. 에센스 카드

drop에서 읽어낸 취향 에센스.

### 구조

```
┌─────────────────────────────┐
│                             │
│  "손 닿는 곳의 아늑함을       │
│   모으는 사람"               │
│                             │
│  [공간] [혼자] [속도]        │
│                             │
│  [차분한] [관조적] [느린]    │
│                             │
└─────────────────────────────┘
```

- 에센스 한 줄 크게 가운데
- 주제 태그 칩들 가로 나열
- 스타일 태그 칩들 가로 나열 (이탤릭)
- 바, 축 없음

### 상태

| 상태 | 설명 |
|------|------|
| 생성 중 | 로딩 → 텍스트 페이드인 |
| 완성 | 에센스 한 줄 + 주제 태그 + 스타일 태그 |
| 축소 | 리스트에서 — 에센스 한 줄만 |

### 맥락별 컬러

| 맥락 | 카드 배경 |
|------|----------|
| 첫 에센스 | 연한 뮤트 톤 블록 |
| 에센스 고도화 | 조금 더 선명한 뮤트 톤 |
| 에센스 변화 감지 | 형광 악센트 (살짝) |

### 디자인 토큰

| 토큰 | 값 |
|------|-----|
| 에센스 텍스트 | Noto Serif KR 24-28px/500 |
| 주제 태그 | Pretendard 12px/400, 뮤트 배경 |
| 스타일 태그 | Pretendard 12px/400, 뮤트 배경, 이탤릭 |
| 카드 패딩 | 24px |
| 카드 라운드 | 8px |

## 3. 인사이트 블록

메타대화에서 발견된 인사이트.

### 구조

```
┌─────────────────────────────┐
│                             │
│  "나는 만드는 건 잘하는데     │
│   닫는 게 안 돼"             │
│                             │
│  #Closer  #통제  #패턴       │
│                             │
└─────────────────────────────┘
```

- 사용자 원문 그대로 표시
- 태그 하단 배치
- 에센스 연결 있으면 하단 한 줄. 없으면 비움

### 상태

| 상태 | 블록 배경 |
|------|----------|
| 신규 | 뮤트 톤 블록 |
| 읽음 | 미색 (기본 배경과 동일) |
| 패턴 연결 | 좌측에 블랙 세로선 1~1.5px |

### 디자인 토큰

| 토큰 | 값 |
|------|-----|
| 발견 텍스트 | Pretendard 16px/400 |
| 태그 | Pretendard 12px/400, 뮤트, 배경 없음 |
| 에센스 연결 | Pretendard 13px/300, 뮤트 |
| 블록 패딩 | 20px |
| 블록 라운드 | 8px |

## 5. 타임라인

시간순 변화 표시. 세로 스크롤.

### 구조

가는 세로선(1~1.5px) 위에 노드가 놓인다. 위가 현재, 아래가 과거.

### 노드 유형

| 유형 | 표시 |
|------|------|
| 에센스 변화 | 에센스 한 줄 비교 (before → after) |
| 패턴 발견 | 태그 + 설명 한 줄 |
| 돌파구 | before 문장 → after 문장. 강조 블록 |
| 첫 에센스 | 에센스 카드 축소형 |

### 맥락별 컬러

- 일반 노드: 미색 배경
- 돌파구 노드: 뮤트 톤 블록
- 변화 감지 노드: 형광 악센트 (살짝)

### 디자인 토큰

| 토큰 | 값 |
|------|-----|
| 세로선 | 블랙, 1~1.5px |
| 날짜 | Pretendard 12px/400, 뮤트 |
| 노드 텍스트 | Pretendard 15px/400 |
| before 텍스트 | Pretendard 14px/300, 뮤트, 취소선 |
| after 텍스트 | Pretendard 14px/400 |
| 노드 마커 | 블랙 원 6px (기본), 10px (돌파구) |

## 7. 입력 영역

drop 입력. 사진, 텍스트, 링크.

### 구조

```
┌─────────────────────────────┐
│  텍스트 입력                  │
│  "요즘 저장만 해둔 글 있어요?" │
│                             │
│  📷  🔗            [drop]   │
└─────────────────────────────┘
```

- 플레이스홀더 질문형
- drop 버튼: 블랙, 텍스트 "drop"
- 아이콘 사진/링크 두 개만

### 상태

| 상태 | 설명 |
|------|------|
| 비어있음 | 플레이스홀더 표시 |
| 입력 중 | 자동 높이 확장 (최대 5줄) |
| 사진 첨부 | 입력 위에 프리뷰. 넉넉한 크기 |
| 링크 첨부 | 입력 위에 제목 + 도메인 한 줄 |
| 전송 중 | drop 버튼 → 로딩 |

### 디자인 토큰

| 토큰 | 값 |
|------|-----|
| 입력 배경 | 미색보다 살짝 밝은 톤 |
| 입력 텍스트 | Pretendard 15px/400 |
| 플레이스홀더 | Pretendard 15px/300, 뮤트 |
| 아이콘 | 24px, 블랙 |
| drop 버튼 | Pretendard 14px/500, 블랙 |
| 입력 라운드 | 8px |
| 입력 패딩 | 12px 16px |


| 대기 마커 (○) | 뮤트 |
| 펄스 | 1.2초 주기 |
| 단계 간격 | 16px |

## 9. Drop 동기화 모션

다른 채널에서 콘텐츠가 넘어올 때.

### 모션 시퀀스

1. 상단에서 물방울 형태 떨어짐
2. 미색 표면 착지
3. 흡수 → 리스트에 배치

### 원칙

- 여러 개면 순차적으로
- 바운스 없음. 중력 + 흡수
- 전체 0.6~0.8초
- 물방울은 추상적 형태. 캐릭터 아님

### 디자인 토큰

| 토큰 | 값 |
|------|-----|
| 물방울 크기 | 12~16px |
| 물방울 색 | 블랙, opacity 0.3 |
| 떨어짐 duration | 0.3초 |
| 흡수 duration | 0.3~0.5초 |
| 연속 drop 간격 | 0.2초 |

## 10. 세션 카드

매거진 그리드의 개별 카드. 세션 페이지와 홈의 최근 읽기에서 사용.

### 구조

왼쪽(100-120px) + 오른쪽(나머지) 2분할. 높이 160px 고정.

왼쪽 영역:
- 사진이 있으면: 사진 배경 + 좌하단에 날짜 (white, drop-shadow)
- 사진이 없으면: 주제 컬러 배경 + 좌하단에 날짜 (white, 18px)

오른쪽 영역:
- 타입 라벨 (11px, 뮤트)
- 인사이트 텍스트 (15px, 세리프, line-clamp-2)
- 관찰 사실 (12px, 뮤트)
- 태그 칩 (하단 정렬)

### 디자인 토큰

| 토큰 | 값 |
|------|-----|
| 카드 높이 | 160px |
| 왼쪽 영역 너비 | 100px (모바일) / 120px (데스크톱) |
| 카드 border | 1px, #040000/8 |
| 카드 라운드 | 8px |
| hover | border #040000/15 |

## 11. 주제 폴더 카드

/themes에서 사용. 겹쳐 쌓이는 카드.

### 구조

- 상단 컬러 바 (5px, 주제 컬러)
- 헤더: 주제 이름 (17px, medium) + "N개 읽기" + 펼침 버튼
- 펼쳐진 상태: 읽기 목록 (날짜 블록 + 인사이트 + 태그)
- 카드 간 겹침: margin-top -24px, 펼치면 아래 카드 밀려남

### 스택 규칙

- 접힌 상태: 헤더만 보임 (약 60px)
- 위 카드가 아래 카드를 24px 덮음
- 펼치면: 해당 카드 전체 높이 확장 + 아래 카드들 translateY로 밀림
- transition: 300ms ease-out

### 디자인 토큰

| 토큰 | 값 |
|------|-----|
| 카드 배경 | White #FFFFFF |
| 카드 shadow | 0 2px 12px rgba(0,0,0,0.06) |
| 컬러 바 높이 | 5px |
| 카드 라운드 | 16px |
| 스택 겹침 | -24px |
