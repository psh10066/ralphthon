# droppi — IA & 와이어프레임 v4

> 플로우 순서대로 정리. 사용자가 처음 들어와서 축적까지.

---

## 1. 페이지 IA (Information Architecture)

```
droppi
│
├── / ─────────────── 랜딩 (풀스크린, 첫 진입)
│   └── drop ────────→ /result
│
├── /result ────────── 읽기 결과 (AI가 읽은 것)
│   ├── 반응 입력 ──→ /chat (메타대화)
│   └── 스킵 ──────→ /home
│
├── /home ─────────── 홈 (일상 진입점) ◆ 탭1
│   ├── drop ────────→ /result
│   ├── 최근 읽기 탭 → /sessions/:id
│   └── 주제 칩 탭 ─→ /themes/:id
│
├── /sessions ─────── 세션 (전체 읽기 목록) ◆ 탭2
│   └── 카드 탭 ────→ /sessions/:id
│
├── /sessions/:id ─── 세션 상세 (읽기 + 대화 원문)
│
├── /chat ─────────── 대화 (메타대화 진행 중)
│   └── 발견 저장 ──→ 프로필에 축적
│
├── /themes/:id ───── 주제 상세 (한 주제의 읽기 모음)
│
└── /profile ─────── 프로필 (축적 전체) ◆ 탭3
    ├── 에센스
    ├── 이번 달 흐름
    ├── 주제 모음 ──→ /themes/:id
    └── 축적된 발견
```

### 네비게이션 규칙

| 요소 | 표시 | 숨김 |
|------|------|------|
| 상단 헤더 (로고) | /home, /sessions, /profile, /themes/:id, /sessions/:id | / (랜딩) |
| 하단 탭 (홈/세션/프로필) | /home, /sessions, /profile | /, /result, /chat, /sessions/:id, /themes/:id |
| ← 뒤로 | /result, /chat, /sessions/:id, /themes/:id | 나머지 |

#### A3. 읽기 결과 `/result`

> AI가 읽은 것 + 질문. 대화를 시작할 맛이 나야 한다.

읽기 결과는 **관찰 + 분석 + 질문** 3요소가 한 세트.
분석만 있으면 "아 그렇구나" 하고 끝남. 질문이 같이 있어야 "어, 아닌데?" → 대화 시작.

```
┌──────────────────────────────┐
│  ⌇ droppi              ← 닫기│  헤더 (로고 + 닫기)
│──────────────────────────────│
│                              │
│  ┌────────────────────────┐  │
│  │                        │  │  drop한 것
│  │   (사진 / 글 미리보기)   │  │  넉넉한 크기
│  │                        │  │
│  └────────────────────────┘  │
│                              │
│                              │
│  사진 속 카페는 구석 자리였고, │
│  조명이 낮고, 혼자 앉는       │  14px, #707980
│  크기의 테이블이에요.         │  ① 관찰 (사실만)
│                              │
│                              │
│  구석을 일부러 골랐다면,       │
│  등 뒤가 막혀있어야           │  Noto Serif KR 20px
│  안심을 느끼는 건              │  ② 분석 (AI의 해석)
│  아닐까.                     │  "~일 수 있다" 톤
│                              │
│                              │
│  ┌──────┐ ┌──────┐ ┌──────┐ │
│  │ 공간  │ │따뜻한 │ │고요한 │ │  태그 칩 (컬러)
│  └──────┘ └──────┘ └──────┘ │
│                              │
│  (축적 2회차+일 때만)          │
│  지난번 작업실 사진에서도       │  13px, #707980
│  벽 쪽이었는데, 이번에도       │  ③ 축적 연결
│  그렇네요.                    │
│                              │
│ ──────────────────────────── │  1px 구분선
│                              │
│  구석을 일부러 골랐어요,       │  15px, #040000
│  아니면 자리가                │  ④ AI의 질문
│  거기뿐이었어요?              │  대화를 여는 한 마디
│                              │
│  ┌────────────────────────┐  │
│  │ 여기에 답해보세요...     │  │  자유 입력 필드
│  │                    →   │  │  bg-white
│  └────────────────────────┘  │
│                              │
│    공유       저장            │  밑줄 텍스트, #707980
│                              │
│       그냥 넘어갈래요          │  13px, #707980/50
│                              │
│  탭 없음.                    │
└──────────────────────────────┘
```

**읽기 결과의 4단 구조:**

| # | 요소 | 역할 | 톤 |
|---|------|------|-----|
| ① | 관찰 | 입력에서 본 사실 | 담백. "~이에요" |
| ② | 분석 | AI의 해석 | 제안형. "~일 수 있다" |
| ③ | 축적 연결 | 과거 읽기와 패턴 (2회차+) | 관찰형. "이번에도 ~네요" |
| ④ | 질문 | 대화를 여는 한 마디 | 호기심. 열린 질문 |

**②분석이 있으니까 ④질문에 답하고 싶어진다.**
분석 없이 질문만 있으면 "뭘 묻는 건데?"
분석이 있으면 "어, 그런가? 아닌데, 그게 아니라..." → 자연스럽게 대화 시작.

**인터랙션:**
- 자유 입력 → [→] 탭 → /chat 전환 (입력 내용이 첫 사용자 메시지)
- "그냥 넘어갈래요" → 읽기(관찰+분석+태그)만 저장, /home 이동
- "공유" → 카카오/이미지 저장
- "← 닫기" → /home

## 4. 반응형 규칙

### 기본 구조

```
┌── 뷰포트 (1440px 기준) ──────────────────────────────┐
│                                                       │
│    ┌── 콘텐츠 영역 (max-width: 1200px, 센터) ──────┐   │
│    │                                               │   │
│    │   좌우 패딩 40px (데스크톱) / 20px (모바일)      │   │
│    │                                               │   │
│    └───────────────────────────────────────────────┘   │
│                                                       │
└───────────────────────────────────────────────────────┘
```

- **최대 너비: 1200px.** 1440px 뷰포트에서 좌우 120px 여백.
- 1200px 초과 시 센터 정렬 + 좌우 여백 자동.
- fullwidth 금지. 콘텐츠가 화면 끝까지 가지 않는다.

### 브레이크포인트

| 이름 | 범위 | 콘텐츠 너비 | 그리드 |
|------|------|-----------|--------|
| **모바일** | < 768px | 100%, 패딩 20px | 1열 |
| **태블릿** | 768px ~ 1023px | 100%, 패딩 32px | 2열 |
| **데스크톱** | ≥ 1024px | max 1200px, 패딩 40px | 2~3열 |

### 페이지별 반응형

| 페이지 | 모바일 | 태블릿 | 데스크톱 |
|--------|--------|--------|---------|
| **랜딩** | 풀스크린 센터 | 풀스크린 센터 | 풀스크린 센터, max-1200px |
| **읽기 결과** | 1열, 사진 full-width | 1열, 사진 max-600px | 1열, 사진 max-600px, 콘텐츠 센터 |
| **홈** | 세션 카드 1열 | 세션 카드 2열 | 세션 카드 2열, 주제 칩 가로 |
| **세션** | 카드 1열 | 카드 2열 | 카드 3열 |
| **주제 상세** | 카드 1열 | 카드 1열 (넓게) | 카드 2열 |
| **대화** | 버블 max-280px | 버블 max-400px | 버블 max-480px, 대화 영역 max-720px 센터 |
| **프로필** | 1열 | 1열 (넓게) | 2컬럼 (에센스+흐름 / 발견+태그) |

### 타이포 반응형

| 요소 | 모바일 | 태블릿 | 데스크톱 |
|------|--------|--------|---------|
| 랜딩 카피 | 28px | 36px | 40px |
| 에센스 한 줄 | 22px | 26px | 28px |
| 읽기 결과 분석 | 18px | 20px | 22px |
| 본문 | 14px | 15px | 15px |
| 뮤트 라벨 | 12px | 13px | 13px |

### 세션 카드 반응형

| 속성 | 모바일 | 태블릿/데스크톱 |
|------|--------|---------------|
| 카드 높이 | 140px | 160px |
| 왼쪽 영역 너비 | 80px | 120px |
| 날짜 텍스트 | 14px | 18px |
| 인사이트 텍스트 | 14px | 15px |

### 하단 탭 + 상단 헤더

- 하단 탭: max-width 1200px, 센터. 양 옆에 Warm White 배경 이어짐
- 상단 헤더: max-width 1200px, 센터. sticky
- 모바일에서는 전체 너비 사용

### 안티패턴

- 콘텐츠가 1200px를 넘어서 좌우로 퍼지는 것
- 모바일에서 좌우 패딩 20px 미만
- 데스크톱에서 텍스트 한 줄이 80자를 넘는 것 (가독성 저하)
- 그리드 없이 텍스트만 1200px 가로로 펼쳐지는 것
