<HARD-GATE>
이 스킬의 모든 체크리스트가 완료되기 전까지 "완료", "끝", "done", "QA 통과"를 선언할 수 없다.
각 단계는 TaskCreate로 생성하고, 증거(스크린샷/로그)와 함께 완료 처리해야 한다.
에러 수정 후에는 반드시 Phase 1부터 전체 재시작한다.
</HARD-GATE>

# QA Gate 스킬

## Phase 1: 로컬 QA

TaskCreate로 아래 항목을 각각 생성한다.

### 1-1. 로컬 서버 기동
- `npm run dev`로 로컬 서버 시작
- 증거: 서버 기동 확인 로그

### 1-2. 전체 페이지 순회 (빈 데이터)
- Playwright MCP로 모든 페이지/메뉴에 접근
- 데이터가 없는 상태에서 정상 렌더링 확인
- 증거: 각 페이지 스크린샷

### 1-3. 테스트 데이터 생성
- 각 화면에 필요한 데이터를 생성
- 증거: 데이터 생성 확인 (DB 또는 UI)

### 1-4. 전체 페이지 순회 (데이터 있음)
- Playwright MCP로 모든 페이지/메뉴에 재접근
- 모든 버튼, 링크, 폼, 인터랙션을 직접 클릭/실행
- 증거: 각 페이지 스크린샷 + 인터랙션 결과

### 1-5. 반응형 확인
- 모바일 뷰포트(390px)로 모든 페이지 재순회
- 증거: 모바일 뷰포트 스크린샷

### 1-6. 콘솔 에러 확인
- 브라우저 콘솔에 에러가 0건인지 확인
- 증거: 콘솔 로그 캡처

### Phase 1 게이트
- 위 항목 중 하나라도 실패 → 수정 후 **Phase 1 전체 재시작**
- 전부 통과 시에만 Phase 2 진입

## Phase 2: 배포

### 2-1. Vercel 배포
- `vercel --prod --yes`
- 증거: 배포 URL 확인

---

## Phase 3: 배포 QA

### 3-1. 배포 URL 전체 페이지 순회 (빈 데이터)
- Phase 1-2와 동일하게 배포 URL에서 수행
- 증거: 각 페이지 스크린샷

### 3-2. 배포 URL 테스트 데이터 생성 + 전체 순회
- Phase 1-3, 1-4와 동일하게 배포 URL에서 수행
- 증거: 각 페이지 스크린샷 + 인터랙션 결과

### 3-3. 배포 URL 반응형 확인
- Phase 1-5와 동일하게 배포 URL에서 수행
- 증거: 모바일 뷰포트 스크린샷

### 3-4. 배포 URL 콘솔 에러 확인
- Phase 1-6과 동일하게 배포 URL에서 수행
- 증거: 콘솔 로그 캡처

### Phase 3 게이트
- 위 항목 중 하나라도 실패 → 수정 후 **Phase 1부터 전체 재시작**
- 전부 통과 시에만 QA 완료 선언 가능

## 반합리화 목록

아래 생각이 떠오르면 **STOP — 합리화하고 있다는 신호다**.

| 금지 생각 | 현실 |
|-----------|------|
| "사소한 이슈라 넘어가자" | 사소한 이슈는 없다. 전부 수정. |
| "이 페이지는 안 봐도 될 것 같다" | 모든 페이지를 봐야 한다. |
| "로컬에서 됐으니 배포도 되겠지" | 배포 후 재검증 필수. |
| "콘솔 에러는 기능에 영향 없다" | 콘솔 에러도 에러다. |
| "데이터 없는 경우는 특수하다" | 빈 상태가 기본 상태다. 반드시 검증. |
| "이 정도면 충분하다" | 체크리스트 전부 통과해야 충분하다. |
| "시간이 없으니 일부만 확인하자" | 부분 검증은 검증이 아니다. |
| "방금 고친 건 작은 수정이라 재검증 안 해도 된다" | 어떤 수정이든 Phase 1부터 재시작. |
| "스크린샷 없어도 확인은 했다" | 증거 없는 확인은 확인이 아니다. |

## 증거 요구

| 주장 | 필요한 증거 |
|------|------------|
| "QA 통과" | Phase 1~3 전체 TaskCreate 완료 + 각 증거 |
| "모든 메뉴 확인" | 메뉴별 스크린샷 (빈 상태 + 데이터 있는 상태) |
| "에러 없음" | 브라우저 콘솔 로그 캡처 (에러 0건) |
| "배포 완료" | 배포 URL + Playwright로 접속한 스크린샷 |
| "반응형 동작" | 모바일 뷰포트(390px) 스크린샷 |
