## 브라우저 API 스택
| 기능 | 사용 기술 |
|------|-----------|
| 마이크 입력 (STT) | Web Speech API - `SpeechRecognition` |
| AI 음성 출력 (TTS) | Web Speech API - `SpeechSynthesis` |

## 작업 규칙
- 외부 API 호출은 `src/lib/`에 모듈 분리
- API Route에서 try-catch 필수
- Supabase 저장 실패해도 핵심 기능은 동작해야 함
- 커밋 메시지는 한국어
- 테스트: Jest (단위) + Playwright (E2E)

## 구현 규칙 (Ralph 루프 진입 시 필수)
아래 규칙은 어떤 플랜이든 Ralph 루프로 구현을 시작하면 반드시 준수해야 한다.

<HARD-GATE>
빌드 성공은 "완료"가 아니다. 코드 작성과 빌드는 전체 작업의 절반일 뿐이다.
빌드 성공 후 반드시 아래 QA 절차를 끝까지 수행해야 한다.
빌드만 완료하고 사용자에게 "다음에 뭘 할까요?"라고 묻는 것은 금지한다.
autopilot, ralph, team, ultrawork 등 어떤 실행 모드든 이 규칙을 따른다.
</HARD-GATE>

### 반응형
- 모바일 앱 UI에서도 동작하는 반응형으로 구현

### 테스트
- 반드시 테스트 코드로 최대한 검증하면서 진행

### 필수 태스크 템플릿

<HARD-GATE>
어떤 실행 모드(autopilot, ralph, team, ultrawork)든 태스크 목록을 생성할 때,
아래 **필수 태스크를 반드시 포함**해야 한다. 구현 태스크만 만들고 QA/배포 태스크를 빠뜨리는 것을 금지한다.
</HARD-GATE>

구현 관련 태스크들 이후에 반드시 아래 태스크들을 순서대로 포함할 것:

| 순서 | 태스크 | 설명 | blockedBy |
|------|--------|------|-----------|
| N-4 | 빌드 검증 및 테스트 | `npm run build` 성공 + 테스트 통과 확인 | 모든 구현 태스크 |
| N-3 | 로컬 QA - Playwright MCP로 전체 기능 확인 | 모든 페이지·메뉴·기능을 Playwright MCP로 직접 검증 | 빌드 검증 |
| N-2 | Vercel 배포 | 로컬 QA 통과 후 Vercel에 배포 | 로컬 QA |
| N-1 | 배포 QA - Playwright MCP로 배포 URL 전체 기능 재확인 | 배포된 URL에서 모든 기능을 다시 검증 | Vercel 배포 |
| N | 반응형 QA - 모바일 뷰포트(390px) 검증 | 배포 URL에서 모바일 뷰포트로 전체 기능 재확인 | 배포 QA |

- 위 태스크들은 `blockedBy`로 순서를 강제할 것
- 로컬 QA, 배포 QA에서 문제 발견 시 수정 후 **해당 QA 태스크를 처음부터 재실행**

### QA 절차

<HARD-GATE>
아래 QA 절차를 모두 완료하기 전에 "완료", "끝", "done", "QA 통과"를 선언할 수 없다.
각 단계는 TaskCreate로 생성하고, 증거(Playwright 스크린샷/콘솔 로그)와 함께 완료 처리해야 한다.
단계를 건너뛸 수 없으며, 이전 단계 미완료 시 다음 단계 진입을 금지한다.
</HARD-GATE>

1. 개발 완료 후 **로컬에서 Playwright MCP로 모든 세부 기능을 하나하나 전부 확인**
2. 로컬 QA 통과 시 **Vercel에 배포**
3. 배포된 URL을 **다시 Playwright MCP로 접근하여 모든 세부 기능을 하나하나 재확인**
4. 문제가 약간이라도 있으면 수정 후 **QA 절차를 처음부터 다시 진행**
5. **문제가 아예 없어야 종료 가능**

### QA 세부 규칙
- 존재하는 **모든 메뉴와 모든 기능을 직접 눌러보고 검사**할 것
- 에러가 발생해서 고쳤다면, **다시 처음부터 모든 기능을 검사**할 것
- 모든 화면에서 **데이터가 없는 경우와 있는 경우를 모두 검토**하고 성공해야 한다
  - 데이터가 없다면 존재하도록 만들고, 잘 동작하는지 확인

### QA 증거 요구
| 주장 | 필요한 증거 |
|------|------------|
| "QA 통과" | Playwright 실행 로그 + 각 페이지 스크린샷 |
| "모든 메뉴 확인" | 메뉴별 스크린샷 (빈 상태 + 데이터 있는 상태) |
| "에러 없음" | 브라우저 콘솔 로그 캡처 (에러 0건) |
| "배포 완료" | 배포 URL + Playwright로 접속한 스크린샷 |
| "반응형 동작" | 모바일 뷰포트(390px) 스크린샷 |

### QA 반합리화 목록
| 금지 생각 | 현실 |
|-----------|------|
| "사소한 이슈라 넘어가자" | 사소한 이슈는 없다. 전부 수정. |
| "이 페이지는 안 봐도 될 것 같다" | 모든 페이지를 봐야 한다. |
| "로컬에서 됐으니 배포도 되겠지" | 배포 후 재검증 필수. |
| "콘솔 에러는 기능에 영향 없다" | 콘솔 에러도 에러다. |
| "데이터 없는 경우는 특수하다" | 빈 상태가 기본 상태다. 반드시 검증. |
| "이 정도면 충분하다" | 체크리스트 전부 통과해야 충분하다. |
| "시간이 없으니 일부만 확인하자" | 부분 검증은 검증이 아니다. |
